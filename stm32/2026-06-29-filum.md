## 一、项目简介

**Filum** 是一个纯 C99 编写的联邦学习（Federated Learning）库，专为 MCU 级边缘设备设计，通过 LoRa 无线电进行通信。一个运行在 STM32F411 上的 **Shard**（分片）在本地传感器数据上训练模型，通过 LoRa 传输稀疏梯度更新；一个运行在 Linux 或树莓派上的 **Herald**（协调者）协调训练轮次，聚合更新，并将改进后的全局模型发回给各 Shard。

核心理念：**无 Python 运行时、无动态内存分配、无堆使用，在 128KB RAM 预算内运行。**

### 核心特性一览

| 特性 | 描述 |
|------|------|
| 纯 C99 | 无 C++、无 RTOS 依赖、无标准库堆 |
| 零动态分配 | 所有缓冲区编译时静态分配 |
| LoRa 原生 | 线缆格式针对 LoRa 载荷限制设计（SF7-SF12） |
| Q8 稀疏梯度 | Top-k 选择 + Q8 量化，约 5.5× 压缩比 |
| 联邦聚合 | 支持 FedAvg、FedProx、坐标中位数聚合 |
| 差分隐私 | 每分片高斯机制，(ε, δ)-DP 保证与预算追踪 |
| ECDH 加密 | Curve25519 密钥交换 + ChaCha20-Poly1305 认证加密 |
| HAL 抽象 | 通过函数指针结构体保持硬件无关 |
| 可配置内存 | 4 级预设，从 STM32F103 的 2KB 到 Linux Herald 的 1MB |

## 四、核心代码深度分析

### 4.1 HAL 抽象层 —— 函数指针结构体

Filum 的硬件抽象层是其可移植性的核心。通过一个函数指针结构体，Shard 逻辑完全独立于具体硬件：

```c
typedef struct {
    int      (*lora_send)(const uint8_t *buf, uint8_t len);        // LoRa 发送
    int      (*lora_recv)(uint8_t *buf, uint8_t *len, uint32_t ms); // LoRa 接收（带超时）
    int      (*lora_set_sf)(uint8_t sf);                            // 设置扩频因子（可选）
    void     (*sleep_ms)(uint32_t ms);                              // 毫秒级睡眠
    void     (*deep_sleep_rtc)(uint32_t seconds);                  // RTC 深度睡眠
    uint32_t (*get_tick_ms)(void);                                  // 获取系统 tick
    int      (*nvs_write)(uint32_t offset, const void *data, size_t len); // 非易失写
    int      (*nvs_read)(uint32_t offset, void *data, size_t len);  // 非易失读
    void     (*log)(const char *fmt, ...);                          // 日志输出（可选）
} FLHal;
```

**设计要点**：

- `lora_set_sf` 和 `log` 可以为 NULL，Shard 代码中会检查指针有效性。
- `nvs_write`/`nvs_read` 用于持久化模型权重，设备重启后可以恢复上次训练的全局模型。
- `deep_sleep_rtc` 在两轮训练之间深度睡眠，大幅降低功耗——这是边缘设备长时间运行的关键。

### 4.2 Q8 量化 —— float 到 int8 的有损压缩

Q8 量化是 Filum 压缩梯度的第一步，将 32 位浮点梯度转换为 8 位有符号整数：

```c
void fl_quant_vec_q8(const float *src, int8_t *dst,
                     uint16_t n, float *scale_out)
{
    if (scale_out) {
        // 计算向量的 L∞ 范数（最大绝对值）
        float norm = fl_grad_linf(src, n);
        if (norm < 1e-9f) norm = 1.0f;  // 避免除零
        *scale_out = norm;               // 输出缩放因子
        for (uint16_t i = 0; i < n; i++)
            dst[i] = fl_quant_q8(src[i] / norm);  // 归一化后量化
    } else {
        // 无缩放输出（直接量化）
        for (uint16_t i = 0; i < n; i++)
            dst[i] = fl_quant_q8(src[i]);
    }
}
```

`fl_quant_q8` 的核心逻辑（定义在 `fl_quant.h` 中）：

```c
// Q8 量化：将 [-1, 1] 范围的 float 映射到 [-127, 127]
static inline int8_t fl_quant_q8(float v) {
    if (v >= 1.0f)  return 127;
    if (v <= -1.0f) return -127;
    return (int8_t)(v * 127.0f);
}

// Q8 反量化：将 int8 恢复为 float
static inline float fl_dequant_q8(int8_t q) {
    return (float)q / 127.0f;
}
```

**逐段解析**：

1. **L∞ 范数归一化**：`fl_grad_linf` 遍历所有梯度，找到绝对值最大的元素。将所有梯度除以此最大值，使所有值落在 `[-1, 1]` 范围内。

2. **缩放因子传输**：`scale_out` 保存了 L∞ 范数值。接收方需要此值来反量化——反量化时 `float = int8 * scale / 127`。

3. **饱和截断**：`fl_quant_q8` 中 `v >= 1.0f` 和 `v <= -1.0f` 的检查是饱和处理，防止溢出。量化范围为 `[-127, 127]`，避免使用 `-128`（因为反量化时 `-128/127 ≈ -1.008` 会超出范围）。

4. **压缩比**：原始 float32 占 4 字节，量化后 int8 占 1 字节，压缩比 4:1。加上缩放因子（1 个 float32 = 4 字节），对于 58 个参数的梯度，原始 232 字节 → 量化后 42 字节 + 4 字节 scale = 46 字节。

### 4.3 梯度裁剪 —— L2 范数约束

在训练过程中，梯度裁剪防止梯度爆炸：

```c
void fl_grad_clip(float *v, uint16_t n, float max_norm)
{
    float norm = fl_grad_l2(v, n);     // 计算 L2 范数
    if (norm > max_norm) {
        float scale = max_norm / (norm + 1e-7f);  // 计算缩放因子
        for (uint16_t i = 0; i < n; i++)
            v[i] *= scale;              // 等比缩放所有梯度
    }
}
```

**解析**：

- `fl_grad_l2` 计算 L2 范数（欧几里得范数）：`sqrt(sum(v[i]^2))`。
- 如果 L2 范数超过 `max_norm`，所有梯度按 `max_norm / norm` 缩放，使缩放后的 L2 范数恰好等于 `max_norm`。
- `1e-7f` 是数值稳定性保护，防止 norm 为零时除零。
- 梯度裁剪在差分隐私中尤为重要——它限制了每个分片对全局模型的最大影响（敏感度），是 DP 噪声校准的基础。

### 4.4 Top-k 稀疏编码 —— 核心压缩算法

Top-k 稀疏编码是 Filum 最核心的创新之一。只传输梯度中绝对值最大的 k 个元素，大幅减少通信量：

```c
uint16_t fl_sparse_encode(const float    *grad,
                          uint16_t        param_count,
                          uint16_t        topk,
                          float          *residual,
                          FLSparseBuffer *out)
{
    if (topk > FL_SPARSE_MAX_ENTRIES) topk = FL_SPARSE_MAX_ENTRIES;
    if (topk > param_count)           topk = param_count;

    // 计算有效梯度 = 原始梯度 + 上次未发送的残差
    static float eff[FL_MODEL_MAX_PARAMS];
    if (param_count > FL_MODEL_MAX_PARAMS) param_count = FL_MODEL_MAX_PARAMS;
    for (uint16_t i = 0; i < param_count; i++) {
        eff[i] = grad[i];
        if (residual) eff[i] += residual[i];  // 累加残差
    }

    // 使用位图标记已选索引
    static uint8_t selected[FL_MODEL_MAX_PARAMS / 8 + 1];
    memset(selected, 0, (param_count / 8) + 1);
    out->count = 0;

    // 部分选择排序：每次选出绝对值最大的未选元素
    for (uint16_t k = 0; k < topk; k++) {
        float    best_val = 0.0f;
        uint16_t best_idx = 0xFFFF;

        for (uint16_t i = 0; i < param_count; i++) {
            if (selected[i / 8] & (1u << (i % 8))) continue;  // 跳过已选
            float a = eff[i] < 0.0f ? -eff[i] : eff[i];       // 绝对值
            if (best_idx == 0xFFFF || a > best_val) {
                best_val = a;
                best_idx = i;
            }
        }

        if (best_idx == 0xFFFF || best_val < 1e-9f) break;  // 无更多有效梯度

        selected[best_idx / 8] |= (uint8_t)(1u << (best_idx % 8));
        out->entries[out->count].param_index = best_idx;     // 记录索引
        out->entries[out->count].delta_q8    = fl_quant_q8(eff[best_idx]);  // Q8 量化
        out->count++;
    }

    // 更新残差：未选中的梯度保留到下一轮
    if (residual) {
        for (uint16_t i = 0; i < param_count; i++) {
            if (selected[i / 8] & (1u << (i % 8)))
                residual[i] = 0.0f;   // 已发送的清零
            else
                residual[i] = eff[i]; // 未发送的保留为残差
        }
    }

    return out->count;
}
```

**逐段深度解析**：

1. **残差累加机制**：`eff[i] = grad[i] + residual[i]`。上一轮未被选中的梯度（残差）会被累加到当前梯度中。这意味着即使某个参数的梯度本轮较小，经过多轮累积后也可能被选中。这保证了所有参数最终都会被更新，避免信息丢失。

2. **位图选择标记**：`selected[]` 是一个位数组，每个 bit 对应一个参数索引。已选索引对应的 bit 被置 1。这比布尔数组节省 8 倍内存。检查使用 `selected[i / 8] & (1u << (i % 8))`。

3. **部分选择排序**：外层循环 k 次（topk），每次内层遍历所有参数找到最大值。时间复杂度 O(n×k)，对于 n < 4096、k < 200 的 MCU 场景足够快。相比完整排序 O(n×log n)，部分排序避免了不必要的计算。

4. **早终止**：`best_val < 1e-9f` 表示剩余梯度都接近零，提前终止，避免发送无意义的零值条目。

5. **稀疏条目格式**：每个条目包含 `param_index`（uint16_t，2 字节）和 `delta_q8`（int8_t，1 字节），共 3 字节。对于 58 个参数、top-25%（14 个条目），总数据量 = 14 × 3 = 42 字节。

6. **残差更新**：已发送的参数残差清零（已经传达），未发送的保留 `eff[i]`（包含本轮梯度 + 之前残差），等待下一轮累积。

**压缩效率计算**：

```
密集 float 上传：58 参数 × 4 字节 = 232 字节
稀疏 Q8 上传：  14 条目 × 3 字节 + 4 字节 scale = 46 字节
压缩比：232 / 42 ≈ 5.5×
LoRa 分片：1 个 SF7 分片即可容纳
```

### 4.5 稀疏解码与分片重组

```c
void fl_sparse_decode(const FLSparseBuffer *buf,
                      float               *grad,
                      uint16_t             param_count,
                      int                  accumulate)
{
    if (!accumulate)
        memset(grad, 0, param_count * sizeof(float));  // 清零目标数组

    // 将稀疏条目展开为密集梯度
    for (uint16_t i = 0; i < buf->count; i++) {
        uint16_t idx = buf->entries[i].param_index;
        if (idx < param_count)
            grad[idx] += fl_dequant_q8(buf->entries[i].delta_q8);
    }
}
```

**解析**：`accumulate` 参数控制是否累加到已有梯度（用于多分片重组）或覆盖。Herald 接收多个分片后，逐个调用 `fl_sparse_read_fragment` 追加到 buffer，然后调用 `fl_sparse_decode` 一次性展开。

### 4.6 线缆协议 —— LoRa 帧格式

Filum 的线缆协议专为 LoRa 的载荷限制设计：

```c
FLError fl_frame_encode(FLFrame       *frame,
                        FLFrameType    type,
                        uint16_t       shard_id,
                        uint8_t        round_id,
                        uint8_t        frag_index,
                        uint8_t        frag_total,
                        const uint8_t *payload,
                        uint8_t        payload_len)
{
    if (!frame)                       return FL_ERR_INVALID_ARG;
    if (payload_len > FL_PAYLOAD_MAX) return FL_ERR_BUFFER_TOO_SMALL;

    frame->magic      = FL_MAGIC;        // 0x464C ('FL')
    frame->frame_type = (uint8_t)type;
    frame->shard_id   = shard_id;
    frame->round_id   = round_id;
    frame->frag_index = frag_index;
    frame->frag_total = frag_total;
    memset(frame->payload, 0, FL_PAYLOAD_MAX);
    if (payload && payload_len > 0)
        memcpy(frame->payload, payload, payload_len);

    frame->crc16 = frame_crc(frame, payload_len);  // CRC-16/CCITT
    return FL_OK;
}
```

**帧结构**：

```
偏移  大小  字段          说明
────────────────────────────────────────────────────────
 0    2    magic         0x464C ('FL')，帧标识
 2    1    frame_type    帧类型（BEACON/UPDATE/ACK/DELTA...）
 3    2    shard_id      分片 ID（0xFFFF = Herald）
 5    1    round_id      训练轮次 ID（0-255 循环）
 6    1    frag_index    当前分片索引（0-based）
 7    1    frag_total    总分片数（1 = 单分片消息）
 8    2    crc16         CRC-16/CCITT-FALSE 校验
10    N    payload       最多 214 字节（SF7）
```

**CRC-16/CCITT 校验**：

```c
static uint16_t frame_crc(const FLFrame *f, uint8_t payload_len)
{
    uint16_t crc = 0xFFFF;  // CRC-16/CCITT-FALSE 初始值
    const uint8_t *raw = (const uint8_t *)f;

    // 计算头部 8 字节的 CRC
    for (int i = 0; i < 8; i++) {
        crc ^= (uint16_t)raw[i] << 8;
        for (int b = 0; b < 8; b++)
            crc = (crc & 0x8000)
                ? (uint16_t)((crc << 1) ^ 0x1021)  // 多项式 0x1021
                : (uint16_t)(crc << 1);
    }
    // 计算载荷的 CRC
    for (int i = 0; i < payload_len; i++) {
        crc ^= (uint16_t)f->payload[i] << 8;
        for (int b = 0; b < 8; b++)
            crc = (crc & 0x8000)
                ? (uint16_t)((crc << 1) ^ 0x1021)
                : (uint16_t)(crc << 1);
    }
    return crc;
}
```

**解析**：

- 使用 CRC-16/CCITT-FALSE 算法，初始值 `0xFFFF`，多项式 `0x1021`。
- 逐字节处理：先将字节 XOR 到 CRC 高 8 位，然后逐 bit 检查最高位——如果为 1 则左移并 XOR 多项式，否则仅左移。
- 这是一种标准的 CRC-16 实现，可检测所有单/双位错误以及大多数突发错误，适合 LoRa 这种可能受噪声干扰的无线通信。

### 4.7 差分隐私 —— 高斯机制

Filum 在 Shard 端实现差分隐私，在梯度上传前添加高斯噪声：

```c
FLDPConfig fl_dp_config(float epsilon, float delta, float sensitivity) {
    FLDPConfig cfg;
    memset(&cfg, 0, sizeof(cfg));
    cfg.epsilon     = epsilon;
    cfg.delta       = delta;
    cfg.sensitivity = sensitivity;
    cfg.dp_enabled  = 1;

    // 高斯机制 sigma 校准：
    //   sigma = S * sqrt(2 * ln(1.25 / delta)) / epsilon
    // 基于 Dwork & Roth (2014), Theorem A.1
    if (epsilon > 0.0f && delta > 0.0f && sensitivity > 0.0f) {
        float log_term = logf(1.25f / delta);
        cfg.sigma = sensitivity * sqrtf(2.0f * log_term) / epsilon;
    } else {
        cfg.dp_enabled = 0;
    }
    return cfg;
}
```

**公式解析**：

- `epsilon`（ε）：隐私预算，越小越隐私但噪声越大。
- `delta`（δ）：失败概率，表示 DP 保证可能以 δ 的概率被违反。
- `sensitivity`（S）：函数敏感度，此处为梯度的 L2 范数上界（通过梯度裁剪控制）。
- `sigma = S * sqrt(2 * ln(1.25/δ)) / ε`：高斯噪声的标准差。σ 越大噪声越大，隐私保护越强。

噪声应用：

```c
void fl_dp_apply(FLDPConfig *cfg, float *grad, uint16_t n,
                 uint16_t shard_id, uint8_t round_id) {
    if (!cfg->dp_enabled || cfg->sigma <= 0.0f) return;

    // 每分片每轮次独立种子，确保：
    //   - 不同 Shard 噪声不同（防止平均攻击）
    //   - 不同轮次噪声不同（防止重放攻击）
    PCG32 rng;
    uint64_t seed = ((uint64_t)shard_id << 32) ^ ((uint64_t)round_id << 16) ^ 0xDEADBEEFULL;
    pcg32_seed(&rng, seed, 0xC0FFEEC0FFEEULL);

    // 向每个梯度坐标添加 N(0, sigma^2) 噪声
    for (uint16_t i = 0; i < n; i++)
        grad[i] += cfg->sigma * pcg32_normal(&rng);

    cfg->rounds_spent++;
}
```

**随机数生成器 —— PCG32**：

Filum 使用 PCG32（Permuted Congruential Generator）而非简单 LCG，因为 PCG32 具有更好的统计特性：

```c
static uint32_t pcg32_next(PCG32 *rng) {
    uint64_t old = rng->state;
    rng->state   = old * 6364136223846793005ULL + (rng->inc | 1);  // LCG 步进
    uint32_t xsh = (uint32_t)(((old >> 18u) ^ old) >> 27u);  // 异或移位
    uint32_t rot = (uint32_t)(old >> 59u);                     // 旋转量
    return (xsh >> rot) | (xsh << ((-rot) & 31));             // 旋转输出
}
```

**Box-Muller 变换 —— 生成正态分布**：

```c
static float pcg32_normal(PCG32 *rng) {
    float u1 = pcg32_uniform(rng) + 1e-7f;  // 避免 log(0)
    float u2 = pcg32_uniform(rng);
    return sqrtf(-2.0f * logf(u1)) * cosf(6.283185307f * u2);
}
```

**解析**：Box-Muller 变换将两个均匀分布随机数 u1, u2 转换为标准正态分布 N(0,1)。公式为 `Z = sqrt(-2*ln(u1)) * cos(2π*u2)`。乘以 σ 后得到 N(0, σ²) 噪声。

**隐私预算追踪**：

```c
void fl_dp_privacy_spent(const FLDPConfig *cfg,
                         float *epsilon_total, float *delta_total) {
    // 基本组合：ε_total = T × ε_round
    *epsilon_total = (float)cfg->rounds_spent * cfg->epsilon;
    *delta_total   = (float)cfg->rounds_spent * cfg->delta;
}
```

使用基本组合定理（Basic Composition）追踪累计隐私消耗：T 轮训练后总 ε 为 `T × ε_round`。开发者可以调用 `fl_shard_privacy_report` 查询当前消耗，在预算耗尽前停止训练。

### 4.8 Shard 状态机 —— 核心运行逻辑

Shard 的核心是一个状态机，驱动整个联邦学习流程：

```c
FLShardState fl_shard_tick(FLShard *shard)
{
    switch (shard->state) {

    case FL_SHARD_STATE_IDLE: {
        // 如果加密已启用但握手未完成，发送 ECDH 握手
        if (shard->crypto.encrypt_enabled && !shard->crypto.handshake_done) {
            FLHandshakePayload hp;
            memcpy(hp.public_key, shard->crypto.public_key, 32);
            shard_tx_frame(shard, FL_FRAME_HANDSHAKE,
                           (uint8_t*)&hp, (uint8_t)sizeof(hp), 0, 1);
        }
        // 监听 Beacon（或握手 ACK）
        uint8_t rx_len = 0;
        int ret = shard->hal->lora_recv(shard->rx_buf, &rx_len, 500);
        if (ret == FL_HAL_OK && rx_len > 0)
            fl_shard_on_rx(shard, shard->rx_buf, rx_len);
        break;
    }

    case FL_SHARD_STATE_TRAINING: {
        // 快照训练前参数
        static float params_before[FL_MODEL_MAX_PARAMS];
        memcpy(params_before, shard->model->params,
               shard->model->total_params * sizeof(float));

        // 配置训练参数
        FLTrainConfig cfg = {
            .epochs         = shard->beacon.local_epochs,
            .learning_rate  = shard->beacon.learning_rate,
            .grad_clip_norm = 1.0f,
            .loss_type      = FL_LOSS_XENTROPY,
            .topk_ratio     = FL_SHARD_TOPK_RATIO,
        };

        // 执行本地 SGD 训练
        fl_train_run(shard->model, &cfg, fl_user_data_cb, NULL);

        // 计算 delta = params_after - params_before
        for (uint16_t i = 0; i < shard->model->total_params; i++)
            shard->model->grad[i] = shard->model->params[i] - params_before[i];

        shard->state = FL_SHARD_STATE_ENCODING;
        break;
    }

    case FL_SHARD_STATE_ENCODING: {
        // 在稀疏编码前应用差分隐私噪声
        if (shard->dp.dp_enabled) {
            fl_dp_apply(&shard->dp, shard->model->grad,
                        shard->model->total_params,
                        shard->shard_id, shard->current_round);
        }

        // 稀疏编码：Top-k 选择 + Q8 量化
        uint16_t topk = shard->model->total_params / FL_SHARD_TOPK_RATIO;
        if (topk < 1) topk = 1;
        fl_sparse_encode(shard->model->grad,
                         shard->model->total_params,
                         topk,
                         shard->residual,
                         &shard->update_buf);

        shard->tx_frag_sent  = 0;
        shard->tx_frag_total = fl_sparse_fragment_count(&shard->update_buf);
        shard->state         = FL_SHARD_STATE_TX;
        break;
    }

    case FL_SHARD_STATE_TX: {
        if (shard->tx_frag_total == 0) {
            shard->state = FL_SHARD_STATE_AWAIT_ACK;
            break;
        }
        // 每轮 tick 发送一个分片（占空比由外部管理）
        if (shard->tx_frag_sent < shard->tx_frag_total) {
            uint8_t payload[FL_PAYLOAD_MAX];
            uint8_t payload_len = 0;
            fl_sparse_write_fragment(payload, &payload_len,
                                     &shard->update_buf,
                                     shard->tx_frag_sent);

            shard_tx_frame(shard, FL_FRAME_UPDATE,
                           payload, payload_len,
                           shard->tx_frag_sent,
                           shard->tx_frag_total);

            if (shard->tx_frag_sent >= shard->tx_frag_total) {
                shard->state = FL_SHARD_STATE_AWAIT_ACK;
            }
        }
        break;
    }

    case FL_SHARD_STATE_AWAIT_ACK: {
        // 等待 Herald 确认
        uint8_t rx_len = 0;
        int ret = shard->hal->lora_recv(shard->rx_buf, &rx_len,
                                        FL_SHARD_RECV_TIMEOUT_MS);
        if (ret == FL_HAL_OK && rx_len > 0)
            fl_shard_on_rx(shard, shard->rx_buf, rx_len);
        break;
    }

    case FL_SHARD_STATE_RECV_DELTA: {
        // 接收聚合后的全局 delta
        uint8_t rx_len = 0;
        int ret = shard->hal->lora_recv(shard->rx_buf, &rx_len,
                                        FL_SHARD_RECV_TIMEOUT_MS);
        if (ret == FL_HAL_OK && rx_len > 0)
            fl_shard_on_rx(shard, shard->rx_buf, rx_len);
        break;
    }
    }
    return shard->state;
}
```

**状态转换图**：

```
                 Beacon 收到
    IDLE ──────────────────► TRAINING
     ▲                         │
     │                         │ 训练完成
     │                         ▼
  ROUND_CLOSE               ENCODING
     │                         │
     │                         │ 编码完成
     │                         ▼
     │                          TX
     │                         │
     │                         │ 所有分片发送完
     │                         ▼
     │                      AWAIT_ACK
     │                         │
     │                         │ ACK 收到
     │                         ▼
     │                      RECV_DELTA
     │                         │
     │                         │ Delta 接收并应用
     └─────────────────────────┘
```

**深度睡眠策略**：

```c
void fl_shard_sleep(FLShard *shard)
{
    if (shard->state == FL_SHARD_STATE_IDLE) {
        // IDLE 状态：深度睡眠直到下一个 Beacon 窗口
        uint32_t sleep_s = shard->beacon.window_seconds > 60
                           ? shard->beacon.window_seconds - 30
                           : 10;
        if (shard->hal->deep_sleep_rtc)
            shard->hal->deep_sleep_rtc(sleep_s);
    } else {
        // 活跃状态：短暂睡眠
        shard->hal->sleep_ms(100);
    }
}
```

在 IDLE 状态下，Shard 深度睡眠 `window_seconds - 30` 秒（留 30 秒余量确保不错过下一个 Beacon）。这使设备在两轮训练之间（可能间隔数小时）几乎零功耗，仅靠 RTC 唤醒。

### 4.9 训练循环 —— 前向传播与反向传播

```c
float fl_loss_forward(const float *pred, const float *target,
                      float *loss_grad, uint8_t n, FLLossType type)
{
    float loss = 0.0f;
    switch (type) {

    case FL_LOSS_MSE:  // 均方误差
        for (uint8_t i = 0; i < n; i++) {
            float d = pred[i] - target[i];
            loss += d * d;
            loss_grad[i] = 2.0f * d / (float)n;  // MSE 梯度
        }
        loss /= (float)n;
        break;

    case FL_LOSS_XENTROPY: {  // Softmax + 交叉熵（融合实现）
        // 数值稳定：减去最大值防止 exp 溢出
        float max_v = pred[0];
        for (uint8_t i = 1; i < n; i++)
            if (pred[i] > max_v) max_v = pred[i];

        float sum_exp = 0.0f;
        static float softmax[64];
        for (uint8_t i = 0; i < n; i++) {
            softmax[i] = expf(pred[i] - max_v);
            sum_exp += softmax[i];
        }
        for (uint8_t i = 0; i < n; i++) {
            softmax[i] /= sum_exp;
            loss_grad[i] = (softmax[i] - target[i]) / (float)n;
            if (target[i] > 0.5f)
                loss -= logf(softmax[i] + 1e-9f);  // 避免 log(0)
        }
        break;
    }
    }
    return loss;
}
```

**Softmax + 交叉熵融合**：通常 Softmax 和交叉熵是两步操作，但融合后梯度简化为 `softmax[i] - target[i]`，数值更稳定且计算更快。`max_v` 减法是标准的数值稳定技巧，防止 `expf` 溢出。

训练 epoch 循环：

```c
uint16_t fl_train_epoch(FLModel *model, const FLTrainConfig *config,
                        FLDataCallback data_cb, void *ctx,
                        float *intermediates)
{
    static FLSample  sample;
    static float     output[64];
    static float     loss_grad[64];
    uint16_t n_samples = 0;

    fl_model_zero_grad(model);  // 梯度清零

    while (data_cb(ctx, &sample)) {           // 逐样本迭代
        fl_model_forward(model, sample.input, output, intermediates);  // 前向
        fl_loss_forward(output, sample.label, loss_grad,
                        out_features, config->loss_type);               // 损失
        fl_model_backward(model, loss_grad, sample.input, intermediates); // 反向

        // 梯度裁剪
        if (config->grad_clip_norm > 0.0f)
            fl_grad_clip(model->grad, model->total_params,
                         config->grad_clip_norm);

        fl_model_sgd_step(model, config->learning_rate);  // SGD 更新
        fl_model_zero_grad(model);  // 清零梯度（逐样本 SGD）
        n_samples++;
    }
    return n_samples;
}
```

**解析**：Filum 使用逐样本 SGD（而非 mini-batch），因为 MCU 内存有限，无法同时保存多个样本。`static` 关键字确保变量分配在 BSS 段而非栈上，避免栈溢出。

### 4.10 聚合策略 —— FedAvg / FedProx / Median

Herald 端支持三种聚合策略：

```c
void fl_agg_finalize(FLAggregator *agg, float *delta_out)
{
    if (agg->shard_count == 0) {
        memset(delta_out, 0, agg->param_count * sizeof(float));
        return;
    }

    switch (agg->type) {

    case FL_AGG_FEDAVG:
        // 简单平均：最常用的联邦聚合
        for (uint16_t i = 0; i < agg->param_count; i++)
            delta_out[i] = agg->accumulator[i] / (float)agg->shard_count;
        break;

    case FL_AGG_FEDPROX:
        // FedProx：加入近端正则项，减少非 IID 数据导致的漂移
        // delta = avg_delta - mu * (avg_params - global_params)
        for (uint16_t i = 0; i < agg->param_count; i++) {
            float avg = agg->accumulator[i] / (float)agg->shard_count;
            delta_out[i] = avg - agg->fedprox_mu *
                           (avg - agg->global_model[i]);
        }
        break;

    case FL_AGG_MEDIAN: {
        // 坐标中位数：拜占庭鲁棒聚合
        // 对每个参数，收集所有 Shard 的值，排序取中位数
        static float col[FL_AGG_MEDIAN_MAX_SHARDS];
        for (uint16_t i = 0; i < agg->param_count; i++) {
            for (uint8_t s = 0; s < agg->shard_count; s++)
                col[s] = agg->median_buf[s][i];
            qsort(col, agg->shard_count, sizeof(float), cmp_float);
            if (agg->shard_count % 2 == 1)
                delta_out[i] = col[agg->shard_count / 2];  // 奇数取中间
            else
                delta_out[i] = (col[agg->shard_count/2 - 1] +
                                col[agg->shard_count/2]) * 0.5f;  // 偶数取平均
        }
        break;
    }
    }
}
```

**三种策略对比**：

| 策略 | 公式 | 优点 | 缺点 |
|------|------|------|------|
| FedAvg | `δ = mean(δ_i)` | 简单高效，数据均衡时最优 | 对拜占庭节点无鲁棒性 |
| FedProx | `δ = avg - μ*(avg - global)` | 减少非 IID 数据漂移 | 需要调 μ 超参数 |
| Median | `δ_i = median(δ_i^1, ..., δ_i^n)` | 拜占庭鲁棒，最多容忍 (n-1)/2 恶意节点 | 计算开销大，需存储所有 Shard 更新 |

### 4.11 加密通信 —— X25519 + ChaCha20-Poly1305

Shard 发送 UPDATE 帧时的加密流程：

```c
static int shard_tx_frame(FLShard *shard, FLFrameType type,
                          const uint8_t *payload, uint8_t payload_len,
                          uint8_t frag_index, uint8_t frag_total)
{
    FLFrame f;
    fl_frame_encode(&f, type, shard->shard_id, shard->current_round,
                    frag_index, frag_total, payload, payload_len);

    // 如果加密已启用且握手已完成，加密 UPDATE 载荷
    uint8_t tag[FL_CRYPTO_TAG_SIZE] = {0};
    if (shard->crypto.encrypt_enabled && shard->crypto.handshake_done &&
        type == FL_FRAME_UPDATE && payload_len > 0) {

        // 生成 nonce：round_id + frag_index + shard_id
        uint8_t nonce[FL_CRYPTO_NONCE_SIZE];
        fl_crypto_make_nonce(nonce, shard->current_round,
                             frag_index, shard->shard_id);

        // ChaCha20 加密 + Poly1305 认证
        fl_crypto_encrypt(&shard->crypto, f.payload, payload_len, nonce, tag);

        // 将认证标签附加在载荷之后
        if ((uint16_t)payload_len + FL_CRYPTO_TAG_SIZE <= FL_PAYLOAD_MAX)
            memcpy(f.payload + payload_len, tag, FL_CRYPTO_TAG_SIZE);
    }

    uint8_t wire_len = (uint8_t)(FL_HEADER_SIZE + payload_len);
    return shard->hal->lora_send((const uint8_t *)&f, wire_len);
}
```

**解析**：

1. **ECDH 密钥交换**：Shard 启动时发送 `FL_FRAME_HANDSHAKE`（包含 X25519 公钥），Herald 回复 `FL_FRAME_HANDSHAKE_ACK`（包含其公钥）。双方用对方公钥和自己的私钥计算共享密钥。

2. **Nonce 生成**：`nonce = round_id || frag_index || shard_id`，确保每个分片的 nonce 唯一，防止重放攻击。

3. **AEAD**：ChaCha20 提供加密，Poly1305 提供认证。Herald 接收时先验证 Poly1305 标签（`FL_ERR_AUTH` 表示篡改），验证通过后才解密。

4. **选择性加密**：只加密 `FL_FRAME_UPDATE`（包含梯度数据），Beacon/ACK 等控制帧不加密（它们不含敏感数据），减少计算开销。

```c
#include <filum.h>

static FLModel model;

}

    // 初始化模型
    fl_model_init(&model, layers, 2);
```

### 内存预设

| 预设 | 最大参数数 | 约需 RAM | 目标平台 |
|------|-----------|---------|---------|

## 七、项目亮点与适用场景

### 技术亮点

1. **真正的 MCU 级联邦学习**：在 128KB RAM 的 STM32F411 上实现完整的联邦学习流程——前向/反向传播、梯度量化、稀疏编码、差分隐私、加密通信，全部在纯 C99 中完成。

2. **5.5× 通信压缩**：通过 Top-k 稀疏选择 + Q8 量化的组合，将 58 参数的梯度从 232 字节压缩到 42 字节，单包 LoRa 即可传输。残差累积机制确保压缩不丢失信息。

3. **差分隐私保障**：实现了完整的高斯机制 DP，包括 PCG32 高质量随机数生成器、Box-Muller 正态分布变换、sigma 校准公式和隐私预算追踪。每个 Shard 独立添加噪声，防止平均攻击。

4. **端到端加密**：X25519 ECDH 密钥交换 + ChaCha20-Poly1305 AEAD 加密，保护梯度数据不被窃听或篡改。纯 C99 实现，不依赖外部加密库。

5. **HAL 抽象设计**：通过 9 个函数指针的结构体实现完全的硬件解耦。移植到新 MCU 只需实现这些函数，Shard 逻辑代码零修改。

6. **三种聚合策略**：FedAvg（标准）、FedProx（非 IID 鲁棒）、Median（拜占庭鲁棒），覆盖了联邦学习的主要聚合需求。

7. **深度睡眠节能**：两轮训练之间通过 RTC 深度睡眠，设备可在电池供电下运行数周甚至数月。

8. **完善的工程实践**：9 个单元/集成测试、CI/CD 流水线、Codecov 覆盖率追踪、Doxygen 文档、语义化版本管理。

### 适用场景

| 场景 | 说明 |
|------|------|
| 分布式环境监测 | 多个传感器节点本地训练异常检测模型，通过 LoRa 聚合 |
| 农业 IoT | 田间传感器节点训练作物状态预测模型，无需云端 |
| 工业预测性维护 | 边缘设备学习设备故障模式，数据不上传保护隐私 |
| 隐私敏感的 ML | 医疗/金融场景，原始数据不出设备，仅交换梯度 |
| LoRa Mesh 智能 | 低功耗广域网中的分布式学习，无需持续网络连接 |
| MCU AI 研究 | 在资源受限设备上研究联邦学习算法的实验平台 |

### 已知限制

- 加密实现未经独立审计（纯 C99，未使用 libsodium/mbedTLS）
- 最大支持 4096 参数（默认预设），大型模型需要 LARGE 预设
- 不支持模型并行或分层联邦学习
- Herald 端需要 Linux 环境（无法在 MCU 上运行）
- LoRa 通信受限于占空比法规（EU 868MHz 1% 占空比限制）

## 八、总结

Filum 是一个开创性的 MCU 级联邦学习框架。它证明了一个事实：在 128KB RAM、100MHz 的微控制器上，完全可以实现包含前向/反向传播、梯度压缩、差分隐私和端到端加密的完整联邦学习流程。

项目的技术深度令人印象深刻：从 PCG32 随机数生成器到 Box-Muller 正态分布变换，从 Top-k 稀疏选择到 Q8 量化压缩，从高斯机制 DP 到 X25519+ChaCha20-Poly1305 加密——每一层都经过精心设计，在 MCU 资源约束下找到最优解。

对于嵌入式 AI 开发者来说，Filum 提供了一个在 LoRa 网络上部署联邦学习的完整参考实现。它的 HAL 抽象层使得移植到不同 MCU 平台只需实现 9 个函数，而三种聚合策略（FedAvg/FedProx/Median）覆盖了从标准到拜占庭鲁棒的各种需求。随着边缘 AI 的快速发展，这类在资源受限设备上实现隐私保护分布式学习的项目将越来越重要。
