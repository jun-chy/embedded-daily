> 仓库作者：Rompass

## 一、项目简介

**OpenC6 BIOS** 是一个为 ESP32-C6（RISC-V 内核）设计的全开源、模块化「BIOS」固件平台。它彻底改变了传统嵌入式「一个单片机写一个大固件」的开发范式：把硬件初始化与应用逻辑解耦，像 PC/服务器那样，由 BIOS 负责初始化硬件、提供带外管理，并通过一套标准化的系统调用接口（ABI）让独立的 `Payload`（负载程序）可以热插拔、下载并直接在 RAM 或 XIP Flash 中执行。

换句话说，它把「主板 BIOS + 操作系统 + 应用」这套 PC 架构，塞进了一颗 2 美元的单片机里。

**核心特点：**

- **LP-Core 管理引擎（ME）**：独立的低功耗 RISC-V 协处理器，即使主系统崩溃也能监控健康状态、读电源键、喂看门狗、触发紧急热关机。
- **SchedUtil 动态调频**：基于 FreeRTOS 空闲周期实时计算负载，在 80/120/160 MHz 间自适应切换。
- **网络启动（PXE）**：通过 Wi-Fi 拉取裸机 Payload 执行，不磨损 Flash。
- **防变砖 A/B OTA**：BIOS 自身可通过网络升级，由硬件回滚机制保护。
- **复古 Web 设置界面**：AP 模式下提供经典蓝屏 BIOS 风格的网页配置器（AI Tweaker 超频、BOD 电平、温度限值）。
- **标准化 ABI**：Payload 用 `-nostdlib -fPIC` 编译，仅 2-10 KB，却能调用 BIOS 提供的 Wi-Fi、数学、加密引擎。

**典型应用场景：** 物联网设备的「固件即平台」架构、需要带外管理的高可靠嵌入式系统、ESP32-C6 上多应用热加载/切换、学习 PC BIOS 架构在 MCU 上的移植、RISC-V 裸机 Payload 开发框架。

### 3.1 系统调用接口（ABI）：Payload 与 BIOS 的契约

OpenC6 的核心思想是：Payload 不链接 ESP-IDF，而是通过一个函数指针表（ABI）调用 BIOS 提供的能力。`openc6_abi.h` 定义了这个契约：

```c
// 文件：components/boot_manager/include/openc6_abi.h

#pragma once
#include <stdint.h>

// 完整性标记 "C6BI"，校验调用上下文是否合法
#define OPENC6_ABI_MAGIC   0x43364249
#define OPENC6_ABI_VERSION 1

/**
 * 传递给独立 Payload 的应用二进制接口（ABI）。
 * 作为一个安全的跳转表，把 Payload 编译与宿主的静态地址映射解耦。
 */
typedef struct {
    uint32_t magic;    // 必须等于 OPENC6_ABI_MAGIC
    uint32_t version;  // 必须等于 OPENC6_ABI_VERSION

    // ─── 系统 API ───
    void (*sys_reset)(void) __attribute__((noreturn));   // 硬复位回到 BIOS
    void (*set_led_color)(uint8_t r, uint8_t g, uint8_t b); // 板载 RGB 状态灯
    void (*delay_ms)(uint32_t ms);                       // 让出线程给 FreeRTOS 防 WDT
    void* (*malloc)(uint32_t size);                      // 从宿主堆分配
    void (*free)(void* ptr);
    void (*print)(const char *str);                      // 打印到 BIOS 控制台
    uint32_t (*get_random)(void);                        // 硬件真随机数 TRNG
    void (*sha256)(const uint8_t *input, uint32_t len, uint8_t *output); // 硬件 SHA-256

    // ─── 数学函数（无 FPU 系统优化）───
    uint32_t (*math_isqrt)(uint32_t x);                  // 整数平方根
    int32_t (*math_sin_deg)(int32_t angle_deg);          // 定点正弦，输出 ×10000
    int32_t (*math_cos_deg)(int32_t angle_deg);

    // ─── 无线网络接口 ───
    int32_t (*wifi_connect)(const char* ssid, const char* pass);
    int32_t (*wifi_start_ap)(const char* ssid, const char* pass);
    int32_t (*wifi_is_connected)(void);

    // ─── 遥测与堆监控 ───
    uint32_t (*get_free_ram)(void);
    uint32_t (*get_total_ram)(void);
    uint32_t (*get_total_flash)(void);

    // ─── 文件系统接口 ───
    void (*fs_write_file)(const char *name, const uint8_t *data, uint32_t len,
                          uint32_t dir_sector, uint8_t force);
    int32_t (*fs_read_file)(const char *name, uint8_t *dest, uint32_t offset,
                            uint32_t len, uint32_t dir_sector);
    void (*fs_delete)(const char *name, uint32_t dir_sector);
} openc6_abi_t;
```

**代码解析：**

- 这是一个**函数指针表结构体**，BIOS 在加载 Payload 时把一个填好的 `openc6_abi_t` 实例的地址传给 Payload 的入口函数。Payload 通过 `abi->wifi_connect(...)` 这样的调用使用 BIOS 能力。
- `magic` + `version` 做**完整性校验**：Payload 启动时先检查 `abi->magic == OPENC6_ABI_MAGIC`，防止拿到错误/过时的 ABI 表导致函数指针乱跳（这在裸机里是灾难性的）。
- 这种设计让 Payload **完全不依赖 ESP-IDF** 编译，体积只有 2-10 KB；同时 Wi-Fi、SHA-256、TRNG 这些重资产只存在于 BIOS 一份，所有 Payload 共享，极大省 Flash 和 RAM。
- `math_sin_deg` 返回值 ×10000（即 sin(90°)=10000），是经典的**定点三角函数**——ESP32-C6 的 RV32IMAC 没有硬件 FPU，用定点避免引入软浮点库。

### 3.2 LP-Core 管理引擎：带外监控与硬复位

这是整个项目最有意思的部分。ESP32-C6 的 LP-Core（低功耗协处理器）通常只用来做待机唤醒，OpenC6 却让它充当「BMC 带外管理控制器」。`lp_core_main.c` 运行在 LP-Core 上：

```c
// 文件：components/management_engine/ulp/lp_core_main.c

#include "ulp_lp_core_utils.h"
#include "ulp_lp_core_gpio.h"
#include "me_shared.h"

// ─── 电源键 GPIO ───
#define PIN_BTN_GND   3
#define PIN_BTN_SENSE 4

// ─── ESP32-C6 LP 看门狗寄存器（直接裸写）───
#define LP_WDT_BASE         0x600B1C00UL
#define LP_WDT_CONFIG0      (*(volatile uint32_t *)(LP_WDT_BASE + 0x00))
#define LP_WDT_CONFIG1      (*(volatile uint32_t *)(LP_WDT_BASE + 0x04))
#define LP_WDT_FEED         (*(volatile uint32_t *)(LP_WDT_BASE + 0x14))
#define LP_WDT_WPROTECT     (*(volatile uint32_t *)(LP_WDT_BASE + 0x18))
#define LP_WDT_WRITE_KEY    0x50D83AA1UL
#define LP_WDT_STG0_RTC_RST (0x4UL << 28)
#define LP_WDT_EN           (1UL << 31)

#define ME_WDT_TIMEOUT_MS   15000

// ─── 跨核共享内存段（IPC）───
volatile uint32_t me_is_enabled   = 1;
volatile uint32_t me_hp_is_awake  = 0;
volatile uint32_t me_boot_reason  = ME_BOOT_REASON_NONE;
volatile uint32_t me_wdt_counter  = 0;
volatile uint32_t me_cpu_load     = 0;
volatile uint32_t me_target_freq  = 80;
volatile uint32_t me_temperature  = 0;
volatile uint32_t me_throttle_temp  = 55;
volatile uint32_t me_emergency_temp = 75;

// 通过 LP 看门狗传播硬件复位
static void me_hw_reset(uint32_t reason) {
    me_boot_reason = reason;
    ulp_lp_core_delay_us(1000);

    LP_WDT_WPROTECT = LP_WDT_WRITE_KEY;   // 解除写保护
    LP_WDT_FEED = (1UL << 31);
    LP_WDT_CONFIG1 = 100UL;                // 复位节拍数
    uint32_t cfg = LP_WDT_CONFIG0;
    cfg &= ~(0x7UL << 28);
    cfg |= LP_WDT_STG0_RTC_RST;            // 阶段0动作 = RTC 复位
    cfg |= LP_WDT_EN;
    LP_WDT_CONFIG0 = cfg;
    LP_WDT_WPROTECT = 0;                   // 重新上锁
    while (1) { __asm__ volatile("nop"); }
}

int main(void) {
    // 配置电源键：GND 脚拉低，Sense 脚上拉输入
    ulp_lp_core_gpio_init(PIN_BTN_GND);
    ulp_lp_core_gpio_output_enable(PIN_BTN_GND);
    ulp_lp_core_gpio_set_level(PIN_BTN_GND, 0);
    ulp_lp_core_gpio_init(PIN_BTN_SENSE);
    ulp_lp_core_gpio_input_enable(PIN_BTN_SENSE);
    ulp_lp_core_gpio_pullup_enable(PIN_BTN_SENSE);

    uint32_t pwr_press_timer = 0, wdt_last_counter = 0, wdt_stale_ms = 0;
    uint32_t sched_loop_timer = 0, me_uptime_ms = 0;

    while (1) {
        if (me_is_enabled == 0) { ulp_lp_core_delay_us(100000); continue; }
        me_uptime_ms += 10;

        // 1. 智能电源键管理
        if (ulp_lp_core_gpio_get_level(PIN_BTN_SENSE) == 0) {
            if (pwr_press_timer < 10000) pwr_press_timer += 10;
            // 长按 5s → 强制硬复位
            if (me_hp_is_awake == 1 && pwr_press_timer >= 5000)
                me_hw_reset(ME_BOOT_REASON_FORCE_RESET);
        } else {
            if (pwr_press_timer > 0) {
                if (me_hp_is_awake == 0) {
                    // 长按 3s → 进设置；短按 → 正常开机
                    if (pwr_press_timer >= 3000) {
                        me_boot_reason = ME_BOOT_REASON_SETUP;
                        ulp_lp_core_wakeup_main_processor();
                    } else if (pwr_press_timer >= 50) {
                        me_boot_reason = ME_BOOT_REASON_NORMAL;
                        ulp_lp_core_wakeup_main_processor();
                    }
                }
                pwr_press_timer = 0;
            }
        }

        // 2. 主处理器软件看门狗解码器
        if (me_hp_is_awake == 1) {
            if (me_wdt_counter != wdt_last_counter) {
                wdt_last_counter = me_wdt_counter;  // 主核在喂狗
                wdt_stale_ms = 0;
            } else {
                wdt_stale_ms += 10;
                if (wdt_stale_ms >= ME_WDT_TIMEOUT_MS)  // 15s 没喂狗 → 复位
                    me_hw_reset(ME_BOOT_REASON_WDT);
            }
        }

        // 3. SchedUtil 调频 + 热保护
        sched_loop_timer += 10;
        if (sched_loop_timer >= 100) {
            sched_loop_timer = 0;
            // 紧急热关机
            if (me_uptime_ms > 3000 && me_temperature >= me_emergency_temp && me_hp_is_awake == 1)
                me_hw_reset(ME_BOOT_REASON_THERMAL);
            // 负载自适应调频（交给主核执行）
            uint32_t target = 80;
            if (me_temperature < me_throttle_temp) {
                if (me_cpu_load > 75) target = 160;
                else if (me_cpu_load > 30) target = 120;
                else target = 80;
            }
            if (target > me_max_freq) target = me_max_freq;
            me_target_freq = target;
        }
        ulp_lp_core_delay_us(10000);  // 10ms 轮询周期
    }
}
```

**代码解析：**

- **跨核 IPC**：`me_*` 这些 `volatile` 全局变量位于 LP-Core 与 HP-Core 共享的内存段（`me_shared.h` 定义）。HP-Core 把 CPU 负载、温度写进来，LP-Core 读；LP-Core 把目标频率、启动原因写出去，HP-Core 读。这是无锁的共享内存通信，因为两边读写的是不同变量，无竞争。
- **看门狗机制**：HP-Core 周期性递增 `me_wdt_counter`（「喂狗」）。LP-Core 每 10ms 检查一次，如果该计数器 15 秒没变化，说明主系统卡死/死锁，LP-Core 直接触发硬件复位。**这就是「带外管理」的精髓**——监控者与被监控者跑在不同核上，主系统崩溃不影响监控者继续工作。
- **硬件复位实现**：直接裸写 LP 看门狗寄存器。先写 `LP_WDT_WRITE_KEY` 解除寄存器写保护（防误写），配置阶段 0 动作为 RTC 复位，再重新上锁。这种寄存器级编程是 ULP 固件的常见写法。
- **电源键逻辑**：短按（>50ms）正常开机，长按 3s 进 BIOS 设置，长按 5s 强制硬复位——完全复刻 PC 主板的电源键行为，但全在 LP-Core 上用 GPIO 软件计时实现。
- **热保护**：温度超过 `me_emergency_temp`（默认 75°C）直接硬复位，超过 `me_throttle_temp`（55°C）强制降到 80MHz 限频。阈值从 NVRAM 动态读取，可在 Web 设置界面里调。

### 3.3 SchedUtil 动态调频：主核侧的执行者

LP-Core 只算出「目标频率」，实际切频由 HP-Core 上的 `power_governor.c` 执行。它还负责把温度传感器的值写回共享内存：

```c
// 文件：components/power_mgmt/power_governor.c

// ─── 结温监控任务（所有模式常驻）───
static void openc6_temperature_task(void *arg) {
    temperature_sensor_handle_t temp_sensor = NULL;
    // 尝试1：默认量程 20~100°C
    temperature_sensor_config_t temp_cfg = TEMPERATURE_SENSOR_CONFIG_DEFAULT(20, 100);
    esp_err_t err = temperature_sensor_install(&temp_cfg, &temp_sensor);
    // 尝试2：高量程失败则回退 -10~80°C
    if (err != ESP_OK) {
        temp_cfg.range_min = -10; temp_cfg.range_max = 80;
        err = temperature_sensor_install(&temp_cfg, &temp_sensor);
    }
    if (err == ESP_OK) temperature_sensor_enable(temp_sensor);

    while (1) {
        if (temp_sensor != NULL) {
            float tsens_value = 0.0f;
            if (temperature_sensor_get_celsius(temp_sensor, &tsens_value) == ESP_OK)
                ulp_me_temperature = (uint32_t)tsens_value;  // 写入 LP-Core 共享 RAM
        } else {
            ulp_me_temperature = 40;  // 传感器不可用时的兜底值
        }
        vTaskDelay(pdMS_TO_TICKS(2000));  // 每 2s 采样一次
    }
}

// ─── 调频任务（仅 DYNAMIC 模式启用）───
static void openc6_schedutil_task(void *arg) {
    uint32_t last_idle_time = 0, last_total_time = 0, current_applied_freq = 0;

    while (1) {
        TaskStatus_t idle_stats;
        vTaskGetInfo(xTaskGetIdleTaskHandle(), &idle_stats, pdFALSE, eInvalid);
        uint32_t now = esp_timer_get_time();        // 微秒
        uint32_t idle_now = idle_stats.ulRunTimeCounter;

        if (last_total_time > 0) {
            uint32_t delta_total = now - last_total_time;
            uint32_t delta_idle = idle_now - last_idle_time;
            // 计算 CPU 负载 0~100%
            uint32_t load = 0;
            if (delta_total > delta_idle)
                load = 100 - ((delta_idle * 100) / delta_total);

            uint32_t target_freq = 80;
            if (is_me_enabled) {
                // ME 启用：把负载上报给 LP-Core，目标频率由 LP-Core 算
                ulp_me_cpu_load = load;
                target_freq = ulp_me_target_freq;
            } else {
                // Legacy 模式：本地直接决策
                if (load > 75) target_freq = 160;
                else if (load > 30) target_freq = 120;
                else target_freq = 80;
            }

            // 只在频率跳变时才真正切频，减少 PMU 抖动
            if (target_freq != current_applied_freq) {
                esp_pm_config_t pm_cfg = {
                    .max_freq_mhz = target_freq,
                    .min_freq_mhz = target_freq,
                    .light_sleep_enable = false
                };
                esp_pm_configure(&pm_cfg);
                current_applied_freq = target_freq;
                ESP_LOGI(TAG, "Load: %lu%% | Temp: %lu C | New Freq: %lu MHz",
                         load, ulp_me_temperature, target_freq);
            }
        }
        last_total_time = now;
        last_idle_time = idle_now;
        vTaskDelay(pdMS_TO_TICKS(100));  // 100ms 重算
    }
}
```

**代码解析：**

- **负载测量**用 FreeRTOS 的运行时统计：`vTaskGetInfo` 拿到空闲任务的 `ulRunTimeCounter`（空闲任务累计运行时间）。两个采样点之间，`delta_idle / delta_total` 就是空闲占比，`100 - 空闲比` 就是 CPU 负载。这是类 Linux `cpufreq` 的经典做法。
- **双模式设计**：`is_me_enabled` 为真时，主核只负责「上报负载 + 执行 LP-Core 算出的目标频率」；为假（Legacy 模式）时主核自己决策。这体现了「管理引擎可选」的弹性——即便没有 LP-Core 协助，系统也能跑。
- **只在跳变时切频**：`if (target_freq != current_applied_freq)` 避免每 100ms 都调一次 `esp_pm_configure`（PMU 切频有开销），只有真正需要变档才动。
- 温度任务做了一次**量程回退**：芯片温度传感器在不同量程下校准精度不同，先用宽量程（20-100°C），失败再用窄量程（-10-80°C），最后兜底写 40°C——典型的防御式嵌入式编程。

### 3.4 Payload 示例：裸机程序如何调用 BIOS ABI

`tools/example/payload.c` 是一个完整的诊断 Payload，展示如何用 `-nostdlib` 编译却仍能用 Wi-Fi、SHA-256、TRNG：

```c
// 文件：tools/example/payload.c
#include "openc6_abi.h"

// 因为 -nostdlib，连 printf 都没有，自己写微型数字转字符串
static void print_uint32(const openc6_abi_t *abi, uint32_t val) {
    char buf[16];
    int i = 14;
    buf[15] = '\0';
    if (val == 0) { buf[i--] = '0'; }
    else {
        while (val > 0 && i >= 0) {
            buf[i--] = '0' + (val % 10);
            val /= 10;
        }
    }
    abi->print(&buf[i + 1]);
}

// Payload 入口：放在 .text.entry 段，由 BIOS 跳转调用
void __attribute__((section(".text.entry"), noreturn))
payload_main(const openc6_abi_t *abi) {

    // 0. ABI 校验
    if (!abi || abi->magic != OPENC6_ABI_MAGIC || abi->version != OPENC6_ABI_VERSION) {
        while (1) { __asm__ volatile("nop"); }
    }

    // [1] 内存监控
    uint32_t flash_kb = abi->get_total_flash() / 1024;
    uint32_t ram_free_kb = abi->get_free_ram() / 1024;
    abi->print(" -> Total Flash: "); print_uint32(abi, flash_kb); abi->print(" KB\n");

    // [2] 动态分配（用 BIOS 的 malloc/free）
    uint8_t *test_buf = (uint8_t*)abi->malloc(1024);
    if (test_buf) { test_buf[0] = 0xAA; abi->free(test_buf); }

    // [3] 硬件 TRNG + SHA-256
    uint32_t r1 = abi->get_random();
    uint8_t hash[32];
    abi->sha256((const uint8_t*)"OpenC6", 6, hash);

    // [4] 定点数学加速
    uint32_t sq_val = abi->math_isqrt(144);          // = 12
    uint32_t sin_val = (uint32_t)abi->math_sin_deg(90); // = 10000

    // [5] Wi-Fi 连接
    int wifi_res = abi->wifi_connect("Test", "12345678");
    if (wifi_res == 0 && abi->wifi_is_connected())
        abi->set_led_color(0, 255, 0);   // 绿色 = 联网成功
    else
        abi->set_led_color(255, 0, 0);   // 红色 = 失败

    // [6] RGB 闪烁演示
    for (int i = 0; i < 3; i++) {
        abi->set_led_color(255, 0, 0); abi->delay_ms(333);
        abi->set_led_color(0, 255, 0); abi->delay_ms(333);
        abi->set_led_color(0, 0, 255); abi->delay_ms(333);
    }

    abi->delay_ms(3000);
    abi->sys_reset();  // 回到 BIOS
}
```

**代码解析：**

- **`__attribute__((section(".text.entry")))`**：把入口函数放进专门的链接段。Payload 是位置无关代码（`-fPIC`），BIOS 加载后从该段的起始地址跳转执行。`noreturn` 表示它不会返回调用者（结束时调 `sys_reset` 回 BIOS）。
- **ABI 校验**是第一道防线：如果 magic/version 不符，直接死循环不往下跑，避免用错误的函数指针把设备搞挂。
- **自己写 `print_uint32`**：因为 `-nostdlib`，没有标准库的 `printf`/`itoa`，只能手写十进制转换。这正是「Payload 极小化」的代价——换来 2-10KB 的体积。
- 整个 Payload 演示了 ABI 的全部能力：内存查询、动态分配、真随机数、硬件 SHA-256、定点数学、Wi-Fi 连接、LED 控制、延时、复位。一个不到 7KB 的程序就能用到 ESP32-C6 的全部关键外设，因为这些驱动都在 BIOS 里。

### 3.5 网络启动（PXE）与无线 OTA

`pxe_boot.c` 实现了两种网络部署：拉取 Payload 到 Flash 执行（PXE），以及 BIOS 自身的无线升级（OTA）：

```c
// 文件：components/pxe_boot/pxe_boot.c

// 1. 用户 Payload 网络部署到 /downloaded/ 目录
bool pxe_boot_execute(const char* url) {
    esp_http_client_config_t config = { .url = url, .timeout_ms = 5000 };
    esp_http_client_handle_t client = esp_http_client_init(&config);
    esp_http_client_open(client, 0);
    int content_length = esp_http_client_fetch_headers(client);

    // 限制最大 150KB，保护 SRAM
    if (content_length > 150 * 1024) { /* 拒绝 */ }

    uint8_t *h_buf = malloc(content_length);  // RAM 暂存整包
    int total_read = 0;
    char buffer[1024];
    while (total_read < content_length) {
        int read_len = esp_http_client_read(client, buffer, sizeof(buffer));
        memcpy(h_buf + total_read, buffer, read_len);
        total_read += read_len;
    }
    // 写入自研文件系统
    uint16_t target_dir_id = get_or_create_dir("downloaded");
    fs_write_file(get_filename_from_url(url), h_buf, total_read, target_dir_id);
    free(h_buf);
    return true;
}

// 2. 无线 BIOS 自更新（直接流式写分区，只占 1KB RAM）
bool pxe_bios_ota_execute(const char* url) {
    // ... 同样初始化 HTTP 客户端 ...
    const esp_partition_t *update_part = esp_ota_get_next_update_partition(NULL);
    esp_ota_handle_t update_handle = 0;
    esp_ota_begin(update_part, OTA_SIZE_UNKNOWN, &update_handle);

    // 流式边下边写，1024 字节一块，仅消耗 1KB RAM
    char buffer[1024];
    int total_read = 0;
    while (total_read < content_length) {
        int read_len = esp_http_client_read(client, buffer, sizeof(buffer));
        esp_ota_write(update_handle, buffer, read_len);  // 直接写 Flash 分区
        total_read += read_len;
    }
    esp_ota_end(update_handle);
    esp_ota_set_boot_partition(update_part);  // 切到新分区启动
    return true;
}
```

**代码解析：**

- **两种部署策略的内存取舍**：PXE 拉 Payload 时先 `malloc` 整包到 RAM（因为 Payload 要校验/重定位），所以限 150KB；而 BIOS OTA 是把固件直接流式写进 Flash 分区，每次只占 1KB 缓冲区，可以烧几百 KB 的固件——这是嵌入式 OTA 的标准内存友好写法。
- **A/B 分区防变砖**：`esp_ota_get_next_update_partition` 拿到「非活动」分区写入，`esp_ota_set_boot_partition` 切换启动分区。如果新固件启动失败、LP-Core 看门狗触发复位，ESP32 bootloader 会自动回滚到旧分区——配合前面 LP-Core 的看门狗，形成「硬件级」防变砖。
- `get_filename_from_url` 用 `strrchr(url, '/')` 从 URL 提取文件名，简洁实用。

### 4.3 网络部署 Payload（PXE）

1. 在 PC 上 `cd tools && python3 -m http.server 8080` 起本地 HTTP 服务。
2. 给 ESP32-C6 上电，长按电源键 3s（或长按 BOOT 进菜单选第 5 项）进 BIOS 设置。
3. 手机连 `BIOS_SETUP_C6`（密码 `12345678`），打开 `http://192.168.4.1`。
4. 填家里 Wi-Fi 账号密码，PXE Server URL 填 `http://<PC局域网IP>:8080/payload.bin`，点 F10 保存退出。
5. 重启时长按 BOOT 进菜单，选 `[0] Network Boot (PXE)`，BIOS 会联网下载 Payload 并执行。

## 五、项目亮点与适用场景

**架构亮点：**

- **MCU 上的 BMC 思路**：把 LP-Core 改造成带外管理引擎，主系统崩溃仍可监控/复位/热关机，这在 MCU 项目里极为罕见。
- **ABI 解耦**：Payload 不依赖 ESP-IDF，2-10KB 即可用 Wi-Fi/加密/文件系统，实现「固件即平台、应用热加载」。
- **PXE + A/B OTA 双重网络部署**：Payload 走 RAM 暂存 + 文件系统，BIOS 走流式直写分区 + 硬件回滚，各自针对场景优化内存。
- **复古 Web BIOS**：AP 模式下的蓝屏网页配置器，把超频、BOD、温度限值做成可视化的「AI Tweaker」，降低配置门槛。
- **NVRAM 虚拟 CMOS**：用 Flash 模拟 PC 的 CMOS，存配置并支持「清 CMOS」恢复出厂。

**适用场景：** 需要远程带外管理的物联网网关、多应用热切换的 ESP32-C6 设备、RISC-V 裸机程序开发与教学、高可靠嵌入式系统（看门狗 + 热保护 + 防变砖 OTA）、研究 PC BIOS 架构在 MCU 上的移植。

## 六、总结

OpenC6 BIOS 是一个「脑洞很大但工程落地扎实」的项目。它最大的价值不是某个单一功能，而是把**服务器/PC 的「BIOS + 带外管理 + 热加载应用 + 网络部署 + 防变砖升级」整套架构范式**完整搬到了一颗 2 美元的 RISC-V 单片机上。LP-Core 管理引擎、ABI 函数指针表、PXE/A/B OTA、NVRAM 虚拟 CMOS——每一块都对应 PC/服务器世界的一个成熟概念，代码实现也足够工程化（防御式校验、量程回退、流式内存优化）。对想跳出「一个 main.c 写到底」思维、学习系统级嵌入式架构的开发者，这个仓库是非常好的参考。
