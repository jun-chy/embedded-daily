## 项目简介与核心特性一览

DSPi 是一款专为 Raspberry Pi Pico（RP2040）和 Pico 2（RP2350）设计的全功能音频 DSP 固件。它将一块几十元的微控制器板变成了一台专业的 USB 音频处理器，支持多通道均衡器、交叉馈送、响度补偿、S/PDIF 输入、PDM 输出等专业音频处理功能。

| 特性 | 说明 |
|------|------|
| USB 音频 | UAC2 高速 USB 音频接口，支持多通道 |
| 参数均衡器 | 每通道最多 10 频段，支持 8 种滤波器类型 |
| 交叉馈送 | BS2B 算法，3 种预设 + 自定义参数 |
| 响度补偿 | 基于 ISO 226:2003 等响曲线，双缓冲系数表 |
| S/PDIF 输入 | 硬件 S/PDIF 接收，带锁相去抖 |
| PDM 输出 | 1-bit PDM 信号生成，用于 D 类功放 |
| 矩阵混音 | 多通道路由与混音 |
| 音量控制 | 主音量、每通道前级、峰值/削波表 |
| 预设管理 | Flash 存储用户预设 |
| 双平台 | RP2040（定点 Q28）+ RP2350（浮点） |

## 核心代码深度分析

### 1. DSP 滤波器系数计算 — 双精度自适应滤波

DSPi 最核心的代码在于 `dsp_pipeline.c` 中的滤波器系数计算函数。该函数同时支持 RP2040 的定点运算和 RP2350 的浮点运算，并智能选择 Biquad 或 SVF（State Variable Filter）结构。

```c
void dsp_compute_coefficients(EqParamPacket *p, Biquad *bq, float sample_rate) {
    // 0xFF-safe: bypass 字节必须恰好为 1 才旁路
    // 任何其他值（0、0xFF 填充、垃圾数据）都保持频段激活
    bool user_bypass = (p->bypass == 1);
    if (user_bypass || is_filter_flat(p) || sample_rate == 0) {
        bq->bypass = true;
#if PICO_RP2350
        bq->b0 = 1.0f; bq->b1 = 0.0f; bq->b2 = 0.0f;
        bq->a1 = 0.0f; bq->a2 = 0.0f;
        bq->use_svf = false;
#else
        bq->b0 = 1 << FILTER_SHIFT; bq->b1 = 0; bq->b2 = 0;
        bq->a1 = 0; bq->a2 = 0;
#endif
        return;
    }
    bq->bypass = false;
    // 输入参数校验
    if (p->Q < 0.1f) p->Q = 0.1f;
    if (p->Q > 20.0f) p->Q = 20.0f;
    if (p->freq < 10.0f) p->freq = 10.0f;
    if (p->freq > sample_rate * 0.45f) p->freq = sample_rate * 0.45f;
    float A = powf(10.0f, p->gain_db / 40.0f);
```

**逐行解析：**

- `bool user_bypass = (p->bypass == 1);` — 使用严格相等比较而非布尔转换。这是嵌入式安全编程的体现：当主机发送的 bypass 字节被 `0xFF` 填充（常见于旧版 USB 主机或未初始化内存），不会误触发旁路。
- `is_filter_flat(p)` — 检查滤波器是否为平坦响应。对于 Peaking/Shelf 类型，当增益小于 0.01dB 时视为平坦，避免不必要的计算开销。
- `#if PICO_RP2350` / `#else` — 条件编译实现双平台支持。RP2350 使用浮点存储（`1.0f`），RP2040 使用定点 Q28 格式（`1 << FILTER_SHIFT`）。
- 参数钳位：Q 值限制在 [0.1, 20.0]，频率限制在 [10Hz, 0.45×采样率]，防止数学不稳定和混叠。
- `float A = powf(10.0f, p->gain_db / 40.0f);` — 增益转换。使用 `/40.0f` 而非 `/20.0f`，因为 RBJ 公式中 A 的定义是 `10^(gain/40)`（对应振幅平方根）。

### 2. SVF 状态变量滤波器（RP2350 专属优化）

```c
#if PICO_RP2350
    // SVF/Biquad 路径选择 + 路径切换时状态重置
    bool was_svf = bq->use_svf;
    bq->use_svf = (p->freq < (sample_rate / 7.5f));
    if (was_svf != bq->use_svf) {
        bq->s1 = 0.0f; bq->s2 = 0.0f;
        bq->svic1eq = 0.0f; bq->svic2eq = 0.0f;
    }
```

**逐行解析：**

- `bq->use_svf = (p->freq < (sample_rate / 7.5f));` — 频率低于采样率的 1/7.5 时使用 SVF。这是因为传统 Biquad 在低频时系数接近数值极限，SVF 在低频具有更好的数值稳定性和调制灵活性。
- `if (was_svf != bq->use_svf)` — 当滤波器路径切换时（如用户调整频率跨过阈值），清零所有积分器状态。这避免了路径切换时的瞬态噪声（"pop"声），是一个关键的音频质量保护措施。

```c
    if (bq->use_svf) {
        // SVF 系数计算（Simper 方法, "SvfLinearTrapAllOutputs", Cytomic 2021）
        float g = tanf(3.1415926535f * p->freq / sample_rate);
        float k = 1.0f / p->Q;
        switch (p->type) {
            case FILTER_PEAKING:
                k = 1.0f / (p->Q * A);
                break;
            case FILTER_LOWSHELF: {
                float sqrtA = sqrtf(A);
                g = g / sqrtA;
                break;
            }
            case FILTER_HIGHSHELF: {
                float sqrtA = sqrtf(A);
                g = g * sqrtA;
                break;
            }
            default: break;
        }
        float sva1_f = 1.0f / (1.0f + g * (g + k));
        float sva2_f = g * sva1_f;
        float sva3_f = g * sva2_f;
```

**解析：**

- `float g = tanf(3.1415926535f * p->freq / sample_rate);` — 预畸变频率映射。通过 tan 函数将模拟频率映射到数字域，补偿双线性变换的频率扭曲。
- `float k = 1.0f / p->Q;` — SVF 阻尼系数。k = 1/Q 使得 SVF 的频率响应与 RBJ Audio-EQ-Cookbook 中的 Biquad 完全一致。
- Peaking 类型使用 `k = 1.0f / (p->Q * A)`，将增益纳入阻尼计算，确保峰值的 Q 值在增益变化时保持一致。
- `sva1_f = 1.0f / (1.0f + g * (g + k));` — SVF 核心系数，等价于 `1/(1+g²+gk)`。这是 Andrew Simper（Cytomic）2021 年发表的线性陷阱全输出 SVF 公式。

### 3. SVF 实时处理 — 按类型特化的内循环

```c
DSP_TIME_CRITICAL
void dsp_process_channel_block(Biquad * __restrict biquads, float * __restrict samples,
                               uint32_t count, uint8_t channel) {
    uint8_t num_bands = channel_band_counts[channel];
    for (int band = 0; band < num_bands; band++) {
        Biquad *bq = &biquads[band];
        if (bq->bypass) continue;
        if (bq->use_svf) {
            float a1 = bq->sva1, a2 = bq->sva2, a3 = bq->sva3;
            float m0 = bq->svm0, m1 = bq->svm1, m2 = bq->svm2;
            float ic1eq = bq->svic1eq, ic2eq = bq->svic2eq;
            float *sp = samples;
            // 按类型特化：消除内循环中的零乘法
            switch (bq->svf_type) {
                case FILTER_LOWPASS:
                    for (uint32_t i = 0; i < count; i++) {
                        float in = *sp;
                        float v3 = in - ic2eq;
                        float v1 = a1 * ic1eq + a2 * v3;
                        float v2 = ic2eq + a2 * ic1eq + a3 * v3;
                        ic1eq = 2.0f * v1 - ic1eq;
                        ic2eq = 2.0f * v2 - ic2eq;
                        *sp++ = v2;
                    }
                    break;
```

**逐行解析：**

- `DSP_TIME_CRITICAL` — 编译器属性标记，将函数放入 SRAM 的 time-critical 段，避免 Flash 访问延迟（RP2040 从 Flash 执行有额外等待周期）。
- `__restrict` — C99 关键字，告知编译器 `biquads` 和 `samples` 指针不重叠，允许更激进的指令调度和向量化。
- 将系数 `a1, a2, a3, m0, m1, m2` 和状态 `ic1eq, ic2eq` 加载到局部变量 — 这是一个关键优化：局部变量被分配到寄存器，避免了内循环中重复通过指针解引用结构体成员。
- **按类型特化**：`switch (bq->svf_type)` 将滤波器类型判断移到循环外。对于 Lowpass，输出直接等于 `v2`，省去了 `m0*in + m1*v1 + m2*v2` 的完整混合计算（其中 m0=0, m1=0, m2=1）。这种特化在每个采样点节省 2 次乘法，在 48kHz/11 通道/10 频段的配置下，每秒节省超过 1000 万次乘法。

### 4. RP2040 定点乘法优化

```c
#if !PICO_RP2350
DSP_TIME_CRITICAL int32_t fast_mul_q28(int32_t a, int32_t b) {
    int32_t ah = a >> 16;
    uint32_t al = a & 0xFFFF;
    int32_t bh = b >> 16;
    uint32_t bl = b & 0xFFFF;
    int32_t high = ah * bh;
    int32_t mid1 = ah * bl;
    int32_t mid2 = al * bh;
    return (high << 4) + ((mid1 + mid2) >> 12);
}
#endif
```

**逐行解析：**

RP2040 的 Cortex-M0+ 内核没有硬件除法器和单周期乘法器（MULS 是 1 周期但 32x32 位乘法需要多周期）。这个 Q28 定点乘法将 32 位乘法分解为 3 个 16x16 位乘法：

- `ah = a >> 16` / `al = a & 0xFFFF` — 将 32 位操作数拆为高 16 位和低 16 位。
- `high = ah * bh` — 高位×高位，贡献 Q28 结果的高位部分，左移 4 位对齐。
- `mid1 + mid2` — 两个交叉项（高位×低位），右移 12 位截取有效位。
- 最终结果 = `(high << 4) + ((mid1 + mid2) >> 12)` — 组合得到 Q28 格式的乘积。
- 这种分解避免了调用 `__aeabi_lmul` 软件库函数，在 RP2040 上实测可提速 3-4 倍。

### 5. BS2B 交叉馈送算法

```c
void crossfeed_compute_coefficients(CrossfeedState *state, const CrossfeedConfig *config,
                                     float sample_rate) {
    // 预设定义：{截止频率Hz, 馈送dB}
    static const float presets[][2] = {
        { 700.0f,  4.5f },  // Default — 平衡，最受欢迎
        { 700.0f,  6.0f },  // Chu Moy — 更强的空间效果
        { 650.0f,  9.5f },  // Jan Meier — 微妙，自然
    };
    // ...
    // 互补约束计算交叉馈送增益 G
    // feed_db 是电平差：20*log10(direct_dc / cross_dc)
    // 互补约束：direct_dc + cross_dc = 1
    float level_ratio = powf(10.0f, feed_db / 20.0f);
    float G = 1.0f / (1.0f + level_ratio);
    // 低通滤波器（交叉馈送路径）— 单极 IIR
    float x = expf(-2.0f * 3.1415926535f * fc / sample_rate);
    float lp_a0_f = G * (1.0f - x);
    float lp_b1_f = x;
```

**逐行解析：**

- BS2B（Bauer Stereophonic-to-Binaural）是一种用于耳机聆听的交叉馈送算法，模拟扬声器在房间中的声音传播，减少耳机中不自然的立体声分离。
- `level_ratio = powf(10.0f, feed_db / 20.0f);` — 将 dB 电平差转换为线性比值。例如 4.5dB 对应 1.679 倍。
- `G = 1.0f / (1.0f + level_ratio);` — 互补约束计算。确保 DC 处直接信号和交叉信号之和为 1（单位增益），避免低频提升。当 feed_db=4.5 时，G=0.373，直接=0.627。
- `float x = expf(-2.0f * PI * fc / sample_rate);` — 一阶 IIR 低通的反馈系数。这是指数衰减形式，等价于 `exp(-2πFc/Fs)`，控制低通截止频率。

```c
    // 互补混合 with ITD：
    //   direct = input - own_lowpass（无延迟互补）
    //   output = direct + allpass(opp_lowpass)（对侧交叉馈送，带延迟）
    *left  = (in_L - lp_out_L) + ap_out_R;
    *right = (in_R - lp_out_R) + ap_out_L;
```

**解析：**

- `(in_L - lp_out_L)` — 直接路径是输入减去自身低通，这是互补滤波器的数学性质。在 DC 处，`in - G*in = (1-G)*in`，直接路径增益为 `1-G`。
- `ap_out_R` — 对侧通道（右声道）低通后再经过全通滤波器延迟的信号。全通滤波器模拟了头部的声学延迟（ITD，双耳时间差），标准值约 220μs。
- 最终输出 = 直接路径 + 延迟后的对侧交叉信号。低频被混合（模拟低频绕过头部），高频保持分离（模拟头部遮挡效应）。

### 6. ISO 226 响度补偿

```c
static float iso226_spl(float Tf, float af, float Lu, float phon) {
    // 阈值项：(0.4 * 10^((Tf+Lu)/10 - 9))^af
    float B = 0.4f * powf(10.0f, (Tf + Lu) / 10.0f - 9.0f);
    float threshold = powf(B, af);
    // Af = phon 相关项 + 阈值
    float Af = 4.47e-3f * (powf(10.0f, 0.025f * phon) - 1.15f) + threshold;
    if (Af < 1e-10f) Af = 1e-10f;
    // Lp = (10/αf) * log10(Af) - Lu + 94
    return (10.0f / af) * log10f(Af) - Lu + 94.0f;
}
```

**逐行解析：**

这是 ISO 226:2003 国际标准中描述等响曲线的数学公式，将感知响度（phon）转换为该频率下的声压级（SPL）：

- `Tf, af, Lu` — ISO 226 表 1 中定义的频率相关常数。例如 50Hz：Tf=44.0, af=0.432, Lu=-15.9。
- `float B = 0.4f * powf(10.0f, (Tf + Lu) / 10.0f - 9.0f);` — 计算听觉阈值项的底数。Tf 是该频率下的阈值声压级修正值。
- `float threshold = powf(B, af);` — 阈值项，af 是指数非线性项。
- `float Af = 4.47e-3f * (powf(10.0f, 0.025f * phon) - 1.15f) + threshold;` — 组合 phon 相关项和阈值项，得到中间变量 Af。
- `(10.0f / af) * log10f(Af) - Lu + 94.0f;` — 反解得到 SPL。94dB 是 1 phon 的参考声压级。

```c
static float loudness_compensation_db(float Tf, float af, float Lu,
                                       float ref_spl, float effective_phon,
                                       float intensity_pct) {
    if (effective_phon >= ref_spl) return 0.0f;  // 参考电平无需补偿
    float spl_ref = iso226_spl(Tf, af, Lu, ref_spl);
    float spl_eff = iso226_spl(Tf, af, Lu, effective_phon);
    float flat_change = effective_phon - ref_spl;
    float freq_change = spl_eff - spl_ref;
    float compensation = freq_change - flat_change;
    compensation *= (intensity_pct / 100.0f);
    return compensation;
}
```

**解析：**

- 当音量降低时，人耳对低频和高频的感知下降比中频更快（Fletcher-Munson 效应）。此函数计算特定频率需要的补偿量。
- `compensation = freq_change - flat_change;` — 该频率 SPL 的实际变化量减去平坦（1kHz）变化量。差值为正表示该频率衰减更多，需要提升补偿。
- `compensation *= (intensity_pct / 100.0f);` — 按用户设定的强度百分比缩放补偿量。
- 该值随后被转换为 Low Shelf（低频架）和 High Shelf（高频架）Biquad 滤波器的增益，实现频率相关的动态音量补偿。

### 7. 双缓冲响度系数表

```c
LoudnessCoeffs loudness_tables[2][LOUDNESS_VOL_STEPS][LOUDNESS_BIQUAD_COUNT];
LoudnessCoeffs (*loudness_active_table)[LOUDNESS_BIQUAD_COUNT] = NULL;
static uint8_t active_buf = 0;

void loudness_recompute_table(float ref_spl, float intensity_pct, float sample_rate) {
    uint8_t write_buf = 1 - active_buf;  // 写入非活跃缓冲区
    for (int vol_idx = 0; vol_idx < LOUDNESS_VOL_STEPS; vol_idx++) {
        // ... 计算 low_shelf 和 high_shelf 系数 ...
        loudness_tables[write_buf][vol_idx][0] = low_shelf_coeffs;
        loudness_tables[write_buf][vol_idx][1] = high_shelf_coeffs;
    }
    // 原子切换：更新活跃表指针
    active_buf = write_buf;
    loudness_active_table = loudness_tables[active_buf];
}
```

**解析：**

- 双缓冲设计确保系数重计算期间音频处理线程仍读取旧系数表，不会出现半更新的中间状态。
- `loudness_tables[2][...]` — 两个完整的系数表，一个活跃一个待写入。
- `active_buf = write_buf; loudness_active_table = ...;` — 两行赋值构成"原子切换"。虽然不是真正的原子操作，但 `loudness_active_table` 指针的更新是最后一步，确保音频线程要么完全读旧表、要么完全读新表。
- `LOUDNESS_VOL_STEPS` 通常为 61 步（-60dB 到 0dB），每步预计算两个 Biquad 的系数，总计 122 组系数。这种查表方式将实时开销从"每采样计算"降为"每采样查表"，在 RP2040 上至关重要。

## API 使用指南

### USB Vendor 命令（cURL 示例）

DSPi 通过 USB Vendor 请求提供控制接口。以下示例使用 `lsusb` 和 Python 的 `usb` 模块：

```python
import usb.core, usb.util, struct

# 查找 DSPi 设备
dev = usb.core.find(idVendor=0x1209, idProduct=0xD5C1)
assert dev is not None, "DSPi 设备未找到"

# 读取设备信息
info = dev.ctrl_transfer(0xC0, 0x01, 0, 0, 64)
print(f"固件版本: {info[0]}.{info[1]}")

# 设置 EQ 参数（通道0, 频段1, Peaking, 1000Hz, Q=0.7, +3dB）
param = struct.pack('<BBBfff', 0, 1, 1, 1000.0, 0.7, 3.0)
dev.ctrl_transfer(0x40, 0x10, 0, 0, param)

# 设置主音量 (-10dB = 0x80)
dev.ctrl_transfer(0x40, 0x20, 0x80, 0, None)

# 切换输入源到 S/PDIF
dev.ctrl_transfer(0x40, 0x30, 1, 0, None)
```

## 编译与部署

### 环境搭建

| 组件 | 版本要求 |
|------|----------|
| Pico SDK | 2.0+（含 pico-extras） |
 | CMake | 3.13+ |
| GCC ARM | 10.3+（arm-none-eabi-gcc） |
| TinyUSB | 随 SDK 内置 |
| Python | 3.6+（用于构建工具） |

### 编译配置表

| CMake 选项 | 默认值 | 说明 |
|------------|--------|------|
| `PICO_PLATFORM` | rp2040 | 目标平台（rp2040/rp2350） |
| `PICO_BOARD` | pico | 板子型号 |
| `DSPi_NUM_CHANNELS` | 7/11 | 通道数（RP2040=7, RP2350=11） |
| `DSPi_MAX_BANDS` | 10 | 每通道最大频段数 |

### 烧录步骤

```bash
# 1. 克隆仓库及子模块
git clone https://github.com/WeebLabs/DSPi.git --recursive
cd DSPi/firmware

# 2. 创建构建目录
mkdir build && cd build

# 3. 配置（RP2040）
cmake -DCMAKE_BUILD_TYPE=Release -DPICO_PLATFORM=rp2040 ..

# 或配置（RP2350 / Pico 2）
cmake -DCMAKE_BUILD_TYPE=Release -DPICO_PLATFORM=rp2350 ..

# 4. 编译
make -j$(nproc) DSPi

# 5. 烧录（按住 BOOTSEL 键，连接 USB）
cp DSPi.uf2 /media/$USER/RPI-RP2/
```

## 项目亮点与适用场景

### 项目亮点

1. **双平台自适应**：同一套代码同时支持 RP2040（定点 Q28）和 RP2350（浮点 SVF），通过条件编译实现零开销抽象。
2. **专业级 DSP 算法**：实现了 RBJ Audio-EQ-Cookbook 全套滤波器、Cytomic SVF、BS2B 交叉馈送、ISO 226 响度补偿，均为业界标准算法。
3. **极致优化**：按类型特化的 SVF 内循环、定点乘法分解、Flash 段时间关键代码，将 RP2040 的性能发挥到极致。
4. **安全性设计**：0xFF-safe bypass、参数钳位、双缓冲系数表、路径切换状态重置，每个细节都考虑了嵌入式边界条件。

### 适用场景

- USB DAC + 参数均衡器
- 耳机放大器（带 BS2B 交叉馈送）
- 有源监听音箱数字分频
- 实时音频效果处理器
- 嵌入式音频教学参考实现

DSPi 是目前 RP2040/RP2350 平台上最完整的开源音频 DSP 固件。它不仅在算法实现上达到了专业标准（RBJ 滤波器、SVF、BS2B、ISO 226 响度补偿），更在工程优化上展现了深厚的嵌入式功底——从定点乘法分解到 SVF 内循环特化，每个优化都直击 RP2040 的性能瓶颈。对于想要学习嵌入式音频处理的开发者来说，这是一个不可多得的参考实现。
