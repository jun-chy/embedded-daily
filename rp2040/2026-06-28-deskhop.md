## 一、项目简介

DeskHop 是一个基于双 Raspberry Pi Pico（RP2040）的硬件 KVM 切换器，允许使用一套键盘鼠标无缝控制两台不同的电脑。用户只需将鼠标指针拖出屏幕边缘，即可在两台电脑之间切换，键盘自动跟随。设备支持 Linux、macOS 和 Windows，切换几乎无延迟，无需安装任何软件，硬件成本低于 15 欧元。

### 解决的问题

作者日常使用两台不同操作系统的电脑，共享一套键盘鼠标。市售 USB 切换器存在切换慢、按键难找、体验差等问题。DeskHop 通过鼠标指针跨屏移动 + 键盘热键两种方式实现无缝切换，甚至跨不同操作系统也能工作。

### 核心特性一览

| 特性 | 说明 |
|------|------|
| 无缝鼠标切换 | 鼠标拖过屏幕边缘自动切换到另一台电脑 |
| 键盘热键 | 自定义快捷键切换（默认 Ctrl + Caps Lock） |
| 无感知延迟 | 切换在毫秒级完成 |

| 屏幕保护 | Pong 球或抖动模式防止电脑休眠 |

| 固件升级 | 支持 OTA 升级，两板互升级 |

## 二、硬件架构

### 2.1 系统组成

```
┌─────────────┐         ┌─────────────────┐         ┌─────────────┐
│  电脑 A     │◄──USB──►│   Board A (Pico) │         │  电脑 B     │
│  (键盘端)   │         │  USB Device      │         │  (鼠标端)   │
└─────────────┘         │  + PIO USB Host  │         └─────────────┘
                        │  (接键盘)        │                ▲
                        └──────┬───┬───────┘                │
                               │   │              ┌───────┴────────┐
                        ┌──────┘   └──────────────│ Board B (Pico) │
                        │                         │ USB Device      │
                   ┌────┴──────┐                  │ + PIO USB Host  │
                   │数字隔离器  │◄──── UART ──────►│ (接鼠标)        │
                   │ISO7721DR  │                  └────────────────┘
                   └───────────┘
```

**工作原理**: 两块 Pico 各自连接一台电脑作为 USB Device（模拟键鼠），同时通过 PIO USB Host 连接实际的键盘或鼠标。两板之间通过 UART + 数字隔离器通信，传递 HID 报告和切换指令。

### 2.2 PIO USB — RP2040 的魔法

RP2040 原生只有一个 USB 端口，但通过 PIO（可编程 IO）可以软件实现第二个 USB 端口。这使得每块 Pico 可以同时作为 USB Host（接键鼠）和 USB Device（连电脑）。

```cpp
void pio_usb_host_config(device_t *state) {
    static pio_usb_configuration_t config = PIO_USB_DEFAULT_CONFIG;
    config.pin_dp = PIO_USB_DP_PIN_DEFAULT;  // D+ pin 14, D- pin 15
    
    // Board B 始终使用 Report 模式
    // Board A 默认使用 Boot 协议（可配置）
    if (state->board_role == OUTPUT_B || ENFORCE_KEYBOARD_BOOT_PROTOCOL == 0)
        tuh_hid_set_default_protocol(HID_PROTOCOL_REPORT);
    
    tuh_configure(BOARD_TUH_RHPORT, TUH_CFGID_RPI_PIO_USB_CONFIGURATION, &config);
    tuh_init(1);  // 初始化 TinyUSB Host
}
```

**解析**: PIO USB 使用 GPIO 14 (D+) 和 GPIO 15 (D-) 实现 USB Host。这是基于 [Pico-PIO-USB](https://github.com/sekigon-gonnoc/Pico-PIO-USB) 项目的软件 USB 实现，利用 RP2040 的可编程状态机在硬件层面精确控制 USB 信号时序。

### 3.2 双核任务调度架构

RP2040 的双核被分配不同职责，Core0 处理 USB Device（向电脑输出），Core1 处理 USB Host（从键鼠读取）：

```c
/* ==================== Core 0 ==================== */
int main(void) {
    static task_t tasks_core0[] = {
        [0] = {.exec = &usb_device_task,          .frequency = _TOP()},      // USB设备任务 - 最高优先级
        [1] = {.exec = &kick_watchdog_task,       .frequency = _HZ(30)},     // 看门狗 - 每33ms验证Core1存活
        [2] = {.exec = &process_kbd_queue_task,   .frequency = _HZ(2000)},   // 键盘队列 - 每0.5ms检查
        [3] = {.exec = &process_mouse_queue_task, .frequency = _HZ(2000)},   // 鼠标队列 - 每0.5ms检查
        [4] = {.exec = &process_hid_queue_task,   .frequency = _HZ(1000)},   // HID包发送 - 每1ms
        [5] = {.exec = &process_uart_tx_task,     .frequency = _TOP()},      // UART发送 - 最高优先级
    };
    
    sleep_ms(10);              // 等待硬件稳定
    initial_setup(device);     // 完整初始化
    set_active_output(device, OUTPUT_A);  // 默认输出 A
    
    while (true) {
        for (int i = 0; i < NUM_TASKS; i++)
            task_scheduler(device, &tasks_core0[i]);
    }
}

/* ==================== Core 1 ==================== */
void core1_main() {
    static task_t tasks_core1[] = {
        [0] = {.exec = &usb_host_task,           .frequency = _TOP()},      // USB主机 - 读取键鼠输入
        [1] = {.exec = &packet_receiver_task,    .frequency = _TOP()},      // 接收另一板的串口数据
        [2] = {.exec = &led_blinking_task,       .frequency = _HZ(30)},     // LED闪烁
        [3] = {.exec = &led_sync_task,           .frequency = _HZ(30)},     // LED状态同步
        [4] = {.exec = &screensaver_task,        .frequency = _HZ(120)},    // 屏幕保护（Pong/抖动）
        [5] = {.exec = &firmware_upgrade_task,   .frequency = _HZ(4000)},   // 固件升级检查
        [6] = {.exec = &heartbeat_output_task,   .frequency = _HZ(1)},      // 心跳 - 每秒一次
    };
    
    while (true) {
        // 更新时间戳，让 Core0 知道我们还活着
        device->core1_last_loop_pass = time_us_32();
        for (int i = 0; i < NUM_TASKS; i++)
            task_scheduler(device, &tasks_core1[i]);
    }
}
```

**解析**: 任务调度器基于频率的协作式调度。`_TOP()` 表示尽可能频繁执行（每次循环都调用），`_HZ(2000)` 表示每秒 2000 次。Core0 专注于向目标 PC 输出 HID 报告（低延迟关键路径），Core1 专注于从键鼠读取输入和辅助功能。`core1_last_loop_pass` 时间戳配合 Core0 的看门狗任务，如果 Core1 卡死超过 30Hz 周期，看门狗会触发系统重启。

### 3.3 板子自动识别（Autoprobe）

两块 Pico 烧录完全相同的固件，通过引脚探测自动判断角色：

```c
int board_autoprobe(void) {
    // 探测序列：8 个上拉/下拉值
    const bool probing_sequence[] = {true, false, false, true, 
                                     true, false, true, false};
    const int seq_len = ARRAY_SIZE(probing_sequence);
    
    // 设置 BOARD_A_RX 引脚为输入
    gpio_init(BOARD_A_RX);
    gpio_set_dir(BOARD_A_RX, GPIO_IN);
    
    for (int i = 0; i < seq_len; i++) {
        // 交替设置上拉或下拉电阻
        if (probing_sequence[i])
            gpio_pull_up(BOARD_A_RX);
        else
            gpio_pull_down(BOARD_A_RX);
        
        sleep_ms(3);  // 等待电平稳定
        bool value = gpio_get(BOARD_A_RX);
        gpio_disable_pulls(BOARD_A_RX);
        
        // 如果读到的值与设置的 pull 不同，
        // 说明有外部 IC（数字隔离器）在驱动该引脚 → 我们是 Board A
        if (probing_sequence[i] != value)
            return OUTPUT_A;
    }
    
    // 所有值都跟随 pull 设置，说明引脚悬空 → 我们是 Board B
    return OUTPUT_B;
}
```

**解析**: 这段代码的巧妙之处在于利用了硬件连接的物理差异。Board A 的 RX 引脚连接了数字隔离器 IC 的输出，IC 会主动驱动引脚电平；Board B 的 RX 引脚悬空。通过交替设置上拉/下拉电阻并读取引脚值：

- **Board A**: 引脚被 IC 驱动，读取值不跟随 pull 设置 → 返回 `OUTPUT_A`
- **Board B**: 引脚悬空，读取值始终跟随 pull 设置 → 返回 `OUTPUT_B`

8 次探测序列确保判断可靠，避免随机噪声导致的误判。这种方案让两块板可以烧录完全相同的固件，极大简化了生产部署。

## 四、核心代码深度分析

### 4.1 DMA 加速的 UART 通信

两块 Pico 之间通过 UART 传输 HID 报告和切换指令。使用 DMA 实现零中断的环形缓冲区：

```c
// 对齐的缓冲区（DMA 环形缓冲区需要地址对齐）
uint8_t uart_rxbuf[DMA_RX_BUFFER_SIZE] 
    __attribute__((aligned(DMA_RX_BUFFER_SIZE)));
uint8_t uart_txbuf[DMA_TX_BUFFER_SIZE] 
    __attribute__((aligned(DMA_TX_BUFFER_SIZE)));

static void configure_rx_dma(device_t *state) {
    // 申请两个 DMA 通道：一个接收，一个控制
    state->dma_rx_channel = dma_claim_unused_channel(true);
    state->dma_control_channel = dma_claim_unused_channel(true);
    
    dma_channel_config config = dma_channel_get_default_config(state->dma_rx_channel);
    dma_channel_config control_config = dma_channel_get_default_config(state->dma_control_channel);
    
    // RX 通道：1 字节传输
    channel_config_set_transfer_data_size(&config, DMA_SIZE_8);
    // 控制通道：4 字节传输（用于重置写入地址）
    channel_config_set_transfer_data_size(&control_config, DMA_SIZE_32);
    
    // 读取地址固定（UART 数据寄存器），写入地址递增（缓冲区）
    channel_config_set_read_increment(&config, false);
    channel_config_set_write_increment(&config, true);
    
    // 环形缓冲区：2^10 = 1024 字节
    channel_config_set_ring(&config, true, 10);
    
    // 由 UART RX 数据可用信号触发 DMA
    channel_config_set_dreq(&config, DREQ_UART0_RX);
    
    // 链式触发：RX 通道完成后触发控制通道
    channel_config_set_chain_to(&config, state->dma_control_channel);
    
    dma_channel_configure(
        state->dma_rx_channel, &config,
        uart_rxbuf,             // 写入环形缓冲区
        &uart0_hw->dr,          // 从 UART 数据寄存器读取
        DMA_RX_BUFFER_SIZE,
        false);
    
    // 控制通道：重置 RX 通道的写入地址，实现无限循环
    dma_channel_configure(
        state->dma_control_channel, &control_config,
        &dma_hw->ch[state->dma_rx_channel].al2_write_addr_trig,  // 写入 RX 通道的写地址寄存器
        uart_buffer_pointers,  // 指向缓冲区起始地址
        1,
        false);
    
    dma_channel_start(state->dma_control_channel);  // 启动！
}
```

**解析**: 这是一个精妙的双 DMA 通道设计：

1. **RX 通道**: 从 UART 数据寄存器读取数据，写入 1024 字节环形缓冲区。`channel_config_set_ring(&config, true, 10)` 让写入地址在 1024 字节范围内自动回绕。
2. **控制通道**: RX 通道传输完 1024 字节后通过链式触发启动控制通道。控制通道将 RX 通道的写入地址重置为缓冲区起始地址，然后触发 RX 通道重新开始。

整个过程无需任何 CPU 中断干预，实现了真正的零开销 UART 接收。CPU 只需在需要时从环形缓冲区读取数据。

发送端 DMA 配置：

```c
static void configure_tx_dma(device_t *state) {
    state->dma_tx_channel = dma_claim_unused_channel(true);
    dma_channel_config tx_config = dma_channel_get_default_config(state->dma_tx_channel);
    channel_config_set_transfer_data_size(&tx_config, DMA_SIZE_8);
    // 读取地址递增（从缓冲区读），写入地址固定（写 UART 寄存器）
    channel_config_set_read_increment(&tx_config, true);
    channel_config_set_write_increment(&tx_config, false);
    // UART TX 触发
    channel_config_set_dreq(&tx_config, DREQ_UART0_TX);
    
    dma_channel_configure(
        state->dma_tx_channel, &tx_config,
        &uart0_hw->dr,      // 写入 UART 数据寄存器
        uart_txbuf,         // 从发送缓冲区读
        0,                  // 传输数量暂设为0，发送时再设置
        false);
}
```

### 4.2 鼠标绝对坐标与屏幕切换

DeskHop 的核心创新是将鼠标从相对坐标改为绝对坐标，实现跨电脑无缝切换：

```c
// 检测鼠标移动是否需要触发屏幕切换
enum screen_pos_e is_screen_switch_needed(int position, int offset) {
    if (position + offset < MIN_SCREEN_COORD - global_state.config.jump_threshold)
        return LEFT;   // 超出左边界
    if (position + offset > MAX_SCREEN_COORD + global_state.config.jump_threshold)
        return RIGHT;  // 超出右边界
    return NONE;
}

// 限制鼠标坐标在屏幕范围内
int32_t move_and_keep_on_screen(int position, int offset) {
    if (position + offset < MIN_SCREEN_COORD)
        return MIN_SCREEN_COORD;
    else if (position + offset > MAX_SCREEN_COORD)
        return MAX_SCREEN_COORD;
    return position + offset;
}

// 更新鼠标位置并返回是否需要切换
enum screen_pos_e update_mouse_position(device_t *state, mouse_values_t *values) {
    output_t *current = &state->config.output[state->active_output];
    uint8_t reduce_speed = 0;
    
    // 慢速模式（精细操作）
    if (state->mouse_zoom)
        reduce_speed = MOUSE_ZOOM_SCALING_FACTOR;
    
    // 计算加速后的移动量
    float accel_factor = calculate_mouse_acceleration_factor(
        values->move_x, values->move_y);
    int offset_x = round(values->move_x * accel_factor * 
                         (current->speed_x >> reduce_speed));
    int offset_y = round(values->move_y * accel_factor * 
                         (current->speed_y >> reduce_speed));
    
    // 检测是否需要切换屏幕
    enum screen_pos_e switch_direction = is_screen_switch_needed(
        state->pointer_x, offset_x);
    
    // 更新位置（限制在屏幕内）
    state->pointer_x = move_and_keep_on_screen(state->pointer_x, offset_x);
    state->pointer_y = move_and_keep_on_screen(state->pointer_y, offset_y);
    
    state->mouse_buttons = values->buttons;
    return switch_direction;
}
```

**解析**: `MIN_SCREEN_COORD` 和 `MAX_SCREEN_COORD` 通常为 0 和 32767。设备内部维护鼠标的绝对位置（而非传统相对移动量），当位置超出边界加 `jump_threshold`（防误触余量）时触发切换。

切换到另一台 PC 的完整流程：

```c
void switch_to_another_pc(device_t *state, output_t *output, 
                          int output_to, int direction) {
    // 1. 将鼠标"停放"到当前屏幕边缘
    uint8_t *mouse_park_pos = &state->config.output[state->active_output].mouse_park_pos;
    int16_t mouse_y = (*mouse_park_pos == 0) ? MIN_SCREEN_COORD :  // 停在顶部
                      (*mouse_park_pos == 1) ? MAX_SCREEN_COORD :  // 停在底部
                                               state->pointer_y;   // 保持当前位置
    mouse_report_t hidden_pointer = {.y = mouse_y, .x = MAX_SCREEN_COORD};
    output_mouse_report(&hidden_pointer, state);
    
    // 2. 切换输出目标
    set_active_output(state, output_to);
    
    // 3. 设置新屏幕上的起始位置
    state->pointer_x = (direction == LEFT) ? MAX_SCREEN_COORD : MIN_SCREEN_COORD;
    
    // 4. 按比例缩放 Y 坐标（支持不同分辨率显示器）
    state->pointer_y = scale_y_coordinate(output->number, 1 - output->number, state);
}

// Y 坐标缩放：处理不同分辨率的显示器
int16_t scale_y_coordinate(int screen_from, int screen_to, device_t *state) {
    output_t *from = &state->config.output[screen_from];
    output_t *to   = &state->config.output[screen_to];
    int size_to   = to->border.bottom - to->border.top;
    int size_from = from->border.bottom - from->border.top;
    
    if (size_from == size_to)
        return state->pointer_y;  // 尺寸相同，无需缩放
    
    // 从小屏幕到大屏幕
    if (size_from > size_to) {
        return to->border.top + ((size_to * state->pointer_y) / MAX_SCREEN_COORD);
    }
    
    // 从大屏幕到小屏幕
    if (state->pointer_y < from->border.top)
        return MIN_SCREEN_COORD;
    if (state->pointer_y > from->border.bottom)
        return MAX_SCREEN_COORD;
    return ((state->pointer_y - from->border.top) * MAX_SCREEN_COORD) / size_from;
}
```

**解析**: 切换流程包含 4 个关键步骤：1) 先将鼠标隐藏到屏幕边缘避免用户看到跳跃；2) 切换 USB 输出目标；3) 在新屏幕设置起始 X（左边缘或右边缘）；4) 按显示器分辨率比例缩放 Y 坐标——这确保从 1080p 屏幕拖到 4K 屏幕时，鼠标高度比例一致。

完整的屏幕切换决策逻辑：

```c
void do_screen_switch(device_t *state, int direction) {
    output_t *output = &state->config.output[state->active_output];
    
    // 锁定或游戏模式下禁止切换
    if (state->switch_lock || state->gaming_mode)
        return;
    
    if (output->pos != direction) {
        // 朝向另一台电脑的方向移动
        if (output->screen_index == 1) {
            // 在主屏幕边界 → 切换到另一台电脑
            if (state->mouse_buttons)  // 按住鼠标键时不切换
                return;
            switch_to_another_pc(state, output, 1 - state->active_output, direction);
        } else {
            // 在虚拟桌面间切换（如 macOS Spaces）
            switch_virtual_desktop(state, output, output->screen_index - 1, direction);
        }
    } else if (output->screen_index < output->screen_count) {
        // 朝远离另一台电脑的方向，但有更多虚拟桌面
        switch_virtual_desktop(state, output, output->screen_index + 1, direction);
    }
}
```

### 4.3 鼠标加速算法

实现了基于 2D 移动幅度的鼠标加速曲线：

```c
#define ACCEL_POINTS 7

float calculate_mouse_acceleration_factor(int32_t offset_x, int32_t offset_y) {
    // 加速曲线：7 个控制点
    const struct curve {
        int value;      // 移动幅度阈值
        float factor;   // 加速因子
    } acceleration[ACCEL_POINTS] = {
        {2, 1},      // 幅度≤2 → 1.0x（无加速）
        {5, 1.1},    // 幅度5  → 1.1x
        {15, 1.4},   // 幅度15 → 1.4x
        {30, 1.9},   // 幅度30 → 1.9x
        {45, 2.6},   // 幅度45 → 2.6x
        {60, 3.4},   // 幅度60 → 3.4x
        {70, 4.0},   // 幅度≥70 → 4.0x
    };
    
    if (offset_x == 0 && offset_y == 0)
        return 1.0;
    if (!global_state.config.enable_acceleration)
        return 1.0;
    
    // 计算二维移动幅度（欧几里得距离）
    const float magnitude = sqrtf(
        (float)(offset_x * offset_x) + 
        (float)(offset_y * offset_y)
    );
    
    // 低于最小阈值
    if (magnitude <= acceleration[0].value)
        return acceleration[0].factor;
    // 高于最大阈值
    if (magnitude >= acceleration[ACCEL_POINTS - 1].value)
        return acceleration[ACCEL_POINTS - 1].factor;
    
    // 在控制点间线性插值
    for (int i = 0; i < ACCEL_POINTS - 1; i++) {
        if (magnitude < acceleration[i + 1].value) {
            const float pos = (magnitude - acceleration[i].value) / 
                              (acceleration[i + 1].value - acceleration[i].value);
            return acceleration[i].factor + 
                   pos * (acceleration[i + 1].factor - acceleration[i].factor);
        }
    }
    return 1.0;
}
```

**解析**: 加速曲线有 7 个控制点，慢速移动（幅度 ≤ 2）不加速（1.0x），快速移动（幅度 ≥ 70）加速到 4.0x。使用 2D 欧几里得距离而非单独的 X/Y 分量，使斜向移动的加速感与直线移动一致。相邻控制点间线性插值保证过渡平滑。

热键匹配算法：

```c
// 检查报告中是否包含指定按键
bool key_in_report(uint8_t key, const hid_keyboard_report_t *report) {
    for (int j = 0; j < KEYS_IN_USB_REPORT; j++) {
        if (key == report->keycode[j])
            return true;
    }
    return false;
}

// 检查是否匹配特定热键
bool check_specific_hotkey(hotkey_combo_t keypress, 
                           const hid_keyboard_report_t *report) {
    // 所有指定的修饰键必须按下
    if (keypress.modifier != (report->modifier & keypress.modifier))
        return false;
    // 所有指定的按键必须按下
    for (int n = 0; n < keypress.key_count; n++) {
        if (!key_in_report(keypress.keys[n], report))
            return false;
    }
    return true;
}

// 遍历热键表
hotkey_combo_t *check_all_hotkeys(hid_keyboard_report_t *report, 
                                   device_t *state) {
    for (int n = 0; n < ARRAY_SIZE(hotkeys); n++) {
        if (check_specific_hotkey(hotkeys[n], report))
            return &hotkeys[n];
    }
    return NULL;
}
```

**解析**: 热键系统使用数据驱动设计——添加新热键只需在数组中加一行，无需修改匹配逻辑。`pass_to_os` 字段控制热键是否仍传递给操作系统（如慢速鼠标模式需要操作系统也看到按键）。`acknowledge` 让 LED 闪烁确认用户操作已被接收。

### 4.5 多键盘合并

DeskHop 支持多个键盘同时连接，将所有键盘状态合并为一个报告：

```c
// 将源报告的按键添加到目标报告（去重）
static void add_keys(hid_keyboard_report_t *dest, 
                      const hid_keyboard_report_t *src) {
    for (uint8_t i = 0; i < KEYS_IN_USB_REPORT; i++) {
        uint8_t key = src->keycode[i];
        if (key == 0 || key_in_report(key, dest))
            continue;
        uint8_t *empty_slot = memchr(dest->keycode, 0, KEYS_IN_USB_REPORT);
        if (empty_slot)
            *empty_slot = key;
    }
}

// 合并所有键盘状态
void combine_kbd_states(device_t *state, hid_keyboard_report_t *combined_report) {
    memset(combined_report, 0, sizeof(hid_keyboard_report_t));
    
    // 合并所有本地键盘
    for (uint8_t i = 0; i <= state->max_kbd_idx; i++) {
        combined_report->modifier |= state->local_kbd_states[i].modifier;
        add_keys(combined_report, &state->local_kbd_states[i]);
    }
    
    // 合并远程键盘（来自另一块板）
    combined_report->modifier |= state->remote_kbd_state.modifier;
    add_keys(combined_report, &state->remote_kbd_state);
}
```

**解析**: USB 键盘报告最多支持 6 个同时按下的键加修饰键。当多个键盘同时输入时，`add_keys` 将所有按键合并到一个报告中（去重避免重复）。修饰键（Ctrl/Shift/Alt 等）使用按位或合并——任何一个键盘按下修饰键都会生效。

### 4.6 配置模式与看门狗寄存器

配置模式利用 RP2040 看门狗寄存器在重启后保留数据的特性：

```c
// 检查是否应进入配置模式
bool is_config_mode_active(device_t *state) {
    // 看门狗寄存器在重启后保留（RP2040 datasheet 2.8.1.1）
    bool is_active = (watchdog_hw->scratch[5] == MAGIC_WORD_1 &&
                      watchdog_hw->scratch[6] == MAGIC_WORD_2);
    
    // 读取后清除，下次重启不再进入
    if (is_active)
        watchdog_hw->scratch[5] = 0;
    
    reset_config_timer(state);
    return is_active;
}

// 触发配置模式
void config_enable_hotkey_handler(device_t *state, hid_keyboard_report_t *report) {
    if (!state->config_mode_active) {
        // 写入 magic word 到看门狗 scratch 寄存器
        watchdog_hw->scratch[5] = MAGIC_WORD_1;
        watchdog_hw->scratch[6] = MAGIC_WORD_2;
    }
    release_all_keys(state);
    state->reboot_requested = true;  // 触发重启
}
```

**解析**: RP2040 的看门狗有 8 个 scratch 寄存器，这些寄存器在软件重启后不会清零。按下配置热键时，向 `scratch[5]` 和 `scratch[6]` 写入预定义的 magic word，然后重启。重启后固件检查这两个寄存器，如果匹配则进入配置模式（USB 磁盘 + Web 配置界面），否则正常运行。进入后立即清除 magic word，确保下次正常启动。

### 4.7 锁屏功能（跨系统）

一个热键同时锁定两台电脑，根据操作系统发送不同的锁屏快捷键：

```c
void screenlock_hotkey_handler(device_t *state, hid_keyboard_report_t *report) {
    hid_keyboard_report_t lock_report = {0}, release_keys = {0};
    
    for (int out = 0; out < NUM_SCREENS; out++) {
        switch (state->config.output[out].os) {
            case WINDOWS:
            case LINUX:
                // Win + L
                lock_report.modifier   = KEYBOARD_MODIFIER_LEFTGUI;
                lock_report.keycode[0] = HID_KEY_L;
                break;
            case MACOS:
                // Ctrl + Cmd + Q
                lock_report.modifier   = KEYBOARD_MODIFIER_LEFTCTRL | 
                                         KEYBOARD_MODIFIER_LEFTGUI;
                lock_report.keycode[0] = HID_KEY_Q;
                break;
            default:
                break;
        }
        
        if (BOARD_ROLE == out) {
            // 本地板：直接发送
            queue_kbd_report(&lock_report, state);
            release_all_keys(state);
        } else {
            // 远程板：通过 UART 发送
            queue_packet((uint8_t *)&lock_report, KEYBOARD_REPORT_MSG, KBD_REPORT_LENGTH);
            queue_packet((uint8_t *)&release_keys, KEYBOARD_REPORT_MSG, KBD_REPORT_LENGTH);
        }
    }
}
```

### 4.8 UART 消息处理

两板之间的所有通信通过 UART 消息系统：

```c
// 处理来自另一板的键盘消息
void handle_keyboard_uart_msg(uart_packet_t *packet, device_t *state) {
    hid_keyboard_report_t *report = (hid_keyboard_report_t *)packet->data;
    hid_keyboard_report_t combined_report;
    
    update_remote_kbd_state(state, report);          // 更新远程键盘状态
    combine_kbd_states(state, &combined_report);     // 合并所有键盘
    queue_kbd_report(&combined_report, state);       // 发送到当前电脑
    
    state->last_activity[BOARD_ROLE] = time_us_64();
}

// 处理来自另一板的鼠标消息
void handle_mouse_abs_uart_msg(uart_packet_t *packet, device_t *state) {
    mouse_report_t *mouse_report = (mouse_report_t *)packet->data;
    
    queue_mouse_report(mouse_report, state);
    state->pointer_x = mouse_report->x;       // 同步鼠标位置
    state->pointer_y = mouse_report->y;
    state->mouse_buttons = mouse_report->buttons;
}

// 处理输出切换请求
void handle_output_select_msg(uart_packet_t *packet, device_t *state) {
    state->active_output = packet->data[0];
    if (state->tud_connected)
        release_all_keys(state);  // 切换时释放所有按键
    restore_leds(state);
}

// 处理固件升级消息
void handle_fw_upgrade_msg(uart_packet_t *packet, device_t *state) {
    reset_usb_boot(1 << PICO_DEFAULT_LED_PIN, 0);  // 进入 BOOTSEL 模式
}
```

### 4.9 初始化流程

完整的系统初始化：

```c
void initial_setup(device_t *state) {
    // PIO USB 需要 12MHz 倍数时钟
    set_sys_clock_khz(120000, true);
    
    // 从 Flash 加载持久化配置
    load_config(state);
    
    // 初始化 LED
    gpio_init(GPIO_LED_PIN);
    gpio_set_dir(GPIO_LED_PIN, GPIO_OUT);
    
    // 检查配置模式（通过看门狗寄存器）
    state->config_mode_active = is_config_mode_active(state);
    
    // 自动探测板子角色（Board A 或 B）
    state->board_role = board_autoprobe();
    
    // 初始化 UART
    serial_init();
    
    // 初始化所有队列
    queue_init(&state->kbd_queue, sizeof(hid_keyboard_report_t), KBD_QUEUE_LENGTH);
    queue_init(&state->mouse_queue, sizeof(mouse_report_t), MOUSE_QUEUE_LENGTH);
    queue_init(&state->hid_queue_out, sizeof(hid_generic_pkt_t), HID_QUEUE_LENGTH);
    queue_init(&state->uart_tx_queue, sizeof(uart_packet_t), UART_QUEUE_LENGTH);
    
    // 启动 Core1
    multicore_reset_core1();
    multicore_launch_core1(core1_main);
    
    // 初始化 TinyUSB Device（连电脑）
    tud_init(BOARD_TUD_RHPORT);
    
    // 初始化 PIO USB Host（接键鼠）
    pio_usb_host_config(state);
    
    // 初始化 DMA（UART 收发）
    configure_tx_dma(state);
    configure_rx_dma(state);
    
    // 加载固件版本信息
    state->_running_fw = _firmware_metadata;
    
    // 设置 Core1 初始时间戳
    state->core1_last_loop_pass = time_us_32();
    
    // 启动看门狗（崩溃自动恢复）
    watchdog_enable(WATCHDOG_TIMEOUT, WATCHDOG_PAUSE_ON_DEBUG);
}
```

**解析**: 初始化流程清晰有序：时钟（120MHz）→ 配置加载 → 硬件探测 → 外设初始化（UART/USB/DMA）→ 多核启动 → 看门狗。看门狗的超时时间（`WATCHDOG_TIMEOUT`）需要大于 Core1 任务循环最长时间，否则正常情况下也会触发重启。

## 六、项目亮点与适用场景

### 技术亮点

1. **PIO USB 创新**: 利用 RP2040 可编程 IO 实现 USB Host，无需专用 USB Host 硬件
2. **双板自识别**: 相同固件通过引脚探测自动识别角色，简化生产部署
3. **DMA 环形缓冲**: 双 DMA 通道链式触发实现零中断 UART 通信
4. **绝对坐标系统**: 修改 HID 描述符实现绝对鼠标定位，跨电脑无缝切换
5. **看门狗配置传递**: 利用 scratch 寄存器在重启后保留配置模式标志
6. **多键盘合并**: 支持多个键盘同时连接，合并为一个 HID 报告
7. **跨系统锁屏**: 根据操作系统自动发送对应锁屏快捷键
8. **电气隔离**: 数字隔离器确保两台电脑完全电气隔离

### 适用场景

- **多电脑办公**: 一套键鼠控制两台不同系统的电脑
- **开发环境**: Linux + Windows/macOS 跨平台开发
- **服务器管理**: 通过 KVM 切换管理多台服务器
- **嵌入式学习**: RP2040 双核调度、PIO USB、DMA、TinyUSB 的优秀教材
- **游戏场景**: 游戏模式切换为相对鼠标，解决兼容性问题
