## 项目简介

DeskHop 是一个基于双 RP2040（Raspberry Pi Pico）的硬件 KVM 切换器，允许使用一套键盘鼠标无缝控制两台不同的电脑。用户只需将鼠标指针拖出屏幕边缘，即可在两台电脑之间切换，键盘自动跟随。设备支持 Linux、macOS 和 Windows，切换无延迟，无需安装任何软件，硬件成本低于 15 欧元。

### 核心特性

- **无缝鼠标切换**: 鼠标指针拖过屏幕边缘即切换到另一台电脑
- **键盘热键切换**: 支持自定义快捷键（默认 Ctrl + Caps Lock）
- **无感知延迟**: 切换在毫秒级完成
- **跨平台**: 支持 Linux、macOS、Windows
- **电气隔离**: 双板之间通过数字隔离器实现 galvanic isolation
- **绝对坐标鼠标**: 修改 HID 描述符实现绝对坐标定位
- **游戏模式**: 可切换为相对鼠标模式，解决游戏兼容性
- **3D 打印外壳**: snap-fit 设计，无需螺丝

### 1. 双核任务调度架构

RP2040 的双核被分别分配不同任务，Core0 处理 USB Device 和输出，Core1 处理 USB Host 和辅助功能：

```c
/* Core0: USB 设备端 + 指令处理 */
int main(void) {
    static task_t tasks_core0[] = {
        [0] = {.exec = &usb_device_task,       .frequency = _TOP()},      // USB设备任务，尽可能频繁运行
        [1] = {.exec = &kick_watchdog_task,    .frequency = _HZ(30)},     // 看门狗喂狗，验证Core1存活
        [2] = {.exec = &process_kbd_queue_task,.frequency = _HZ(2000)},   // 处理键盘队列
        [3] = {.exec = &process_mouse_queue_task, .frequency = _HZ(2000)},// 处理鼠标队列
        [4] = {.exec = &process_hid_queue_task,.frequency = _HZ(1000)},   // HID数据包发送
        [5] = {.exec = &process_uart_tx_task,  .frequency = _TOP()},      // UART发送
    };
    
    sleep_ms(10);
    initial_setup(device);
    set_active_output(device, OUTPUT_A);
    
    while (true) {
        for (int i = 0; i < NUM_TASKS; i++)
            task_scheduler(device, &tasks_core0[i]);
    }
}

/* Core1: USB 主机端 + 辅助功能 */
void core1_main() {
    static task_t tasks_core1[] = {
        [0] = {.exec = &usb_host_task,         .frequency = _TOP()},      // USB主机任务
        [1] = {.exec = &packet_receiver_task,  .frequency = _TOP()},      // 接收另一块板的串口数据
        [2] = {.exec = &led_blinking_task,     .frequency = _HZ(30)},     // LED闪烁
        [3] = {.exec = &led_sync_task,         .frequency = _HZ(30)},     // LED状态同步
        [4] = {.exec = &screensaver_task,      .frequency = _HZ(120)},    // 屏幕保护
        [5] = {.exec = &firmware_upgrade_task, .frequency = _HZ(4000)},   // 固件升级
        [6] = {.exec = &heartbeat_output_task, .frequency = _HZ(1)},     // 心跳
    };
    
    while (true) {
        device->core1_last_loop_pass = time_us_32();
        for (int i = 0; i < NUM_TASKS; i++)
            task_scheduler(device, &tasks_core1[i]);
    }
}
```

**解析**: 采用基于频率的协作式任务调度器，每个任务指定执行频率（`_TOP()` 表示尽可能频繁，`_HZ(2000)` 表示每秒 2000 次）。Core0 专注于向目标 PC 输出 HID 报告，Core1 专注于从键鼠读取输入。`core1_last_loop_pass` 时间戳让 Core0 可以检测 Core1 是否卡死，配合看门狗实现崩溃自动恢复。

### 2. 板子自动识别（Autoprobe）

两块 Pico 使用相同的固件，通过引脚探测自动判断自己是 Board A 还是 Board B：

```c
int board_autoprobe(void) {
    const bool probing_sequence[] = {true, false, false, true, 
                                     true, false, true, false};
    const int seq_len = ARRAY_SIZE(probing_sequence);
    
    gpio_init(BOARD_A_RX);
    gpio_set_dir(BOARD_A_RX, GPIO_IN);
    
    for (int i = 0; i < seq_len; i++) {
        if (probing_sequence[i])
            gpio_pull_up(BOARD_A_RX);
        else
            gpio_pull_down(BOARD_A_RX);
        
        sleep_ms(3);
        bool value = gpio_get(BOARD_A_RX);
        gpio_disable_pulls(BOARD_A_RX);
        
        // 如果读到的值与预设不同，说明有 IC 驱动该引脚 → 是 Board A
        if (probing_sequence[i] != value)
            return OUTPUT_A;
    }
    
    // 如果所有值都跟随 pull，说明引脚悬空 → 是 Board B
    return OUTPUT_B;
}
```

**解析**: 这是一个巧妙的硬件识别方案。Board A 的 RX 引脚连接了数字隔离器 IC，IC 会驱动引脚电平；Board B 的 RX 引脚悬空。通过交替设置上拉/下拉电阻并读取引脚值：如果值始终跟随 pull 设置，说明引脚悬空（Board B）；如果值不跟随，说明有 IC 在驱动（Board A）。这样两块板可以烧录完全相同的固件，简化生产和维护。

### 3. DMA 加速的 UART 通信

两块 Pico 之间通过 UART 通信，使用 DMA 实现高效数据传输：

```c
static void configure_rx_dma(device_t *state) {
    state->dma_rx_channel = dma_claim_unused_channel(true);
    state->dma_control_channel = dma_claim_unused_channel(true);
    
    dma_channel_config config = dma_channel_get_default_config(state->dma_rx_channel);
    dma_channel_config control_config = dma_channel_get_default_config(state->dma_control_channel);
    
    channel_config_set_transfer_data_size(&config, DMA_SIZE_8);
    channel_config_set_transfer_data_size(&control_config, DMA_SIZE_32);
    
    // 读取地址固定（UART 数据寄存器），写入地址递增（缓冲区）
    channel_config_set_read_increment(&config, false);
    channel_config_set_write_increment(&config, true);
    // 环形缓冲区，1024 字节（2^10）
    channel_config_set_ring(&config, true, 10);
    // UART RX 触发 DMA
    channel_config_set_dreq(&config, DREQ_UART0_RX);
    // 链式触发：接收完后重启控制通道
    channel_config_set_chain_to(&config, state->dma_control_channel);
    
    dma_channel_configure(
        state->dma_rx_channel, &config,
        uart_rxbuf,             // 写入环形缓冲区
        &uart0_hw->dr,          // 从 UART 数据寄存器读
        DMA_RX_BUFFER_SIZE,
        false);
    
    // 控制通道：重置 RX 通道的写入地址，实现无限循环
    dma_channel_configure(
        state->dma_control_channel, &control_config,
        &dma_hw->ch[state->dma_rx_channel].al2_write_addr_trig,
        uart_buffer_pointers,
        1,
        false);
    
    dma_channel_start(state->dma_control_channel);
}
```

**解析**: 使用两个 DMA 通道配合实现无限循环的 UART 接收：RX 通道将数据写入 1024 字节环形缓冲区，完成后触发控制通道，控制通道重置 RX 通道的写入地址到缓冲区起始位置并重新启动。整个过程无需 CPU 干预，极大地降低了中断负载。

### 4. 鼠标切换核心逻辑

当鼠标指针移动到屏幕边缘时，触发电脑切换：

```c
// 检测是否需要切换屏幕
enum screen_pos_e is_screen_switch_needed(int position, int offset) {
    if (position + offset < MIN_SCREEN_COORD - global_state.config.jump_threshold)
        return LEFT;
    if (position + offset > MAX_SCREEN_COORD + global_state.config.jump_threshold)
        return RIGHT;
    return NONE;
}

// 执行屏幕切换
void do_screen_switch(device_t *state, int direction) {
    output_t *output = &state->config.output[state->active_output];
    
    // 游戏模式或锁定时不切换
    if (state->switch_lock || state->gaming_mode)
        return;
    
    if (output->pos != direction) {
        if (output->screen_index == 1) {
            // 在边界 → 切换到另一台电脑
            if (state->mouse_buttons)  // 按住鼠标键时不切换
                return;
            switch_to_another_pc(state, output, 1 - state->active_output, direction);
        } else {
            // 在虚拟桌面间切换
            switch_virtual_desktop(state, output, output->screen_index - 1, direction);
        }
    } else if (output->screen_index < output->screen_count) {
        switch_virtual_desktop(state, output, output->screen_index + 1, direction);
    }
}

// 切换到另一台 PC
void switch_to_another_pc(device_t *state, output_t *output, 
                          int output_to, int direction) {
    // 将鼠标"停放"到边缘
    uint8_t *mouse_park_pos = &state->config.output[state->active_output].mouse_park_pos;
    int16_t mouse_y = (*mouse_park_pos == 0) ? MIN_SCREEN_COORD : 
                      (*mouse_park_pos == 1) ? MAX_SCREEN_COORD : 
                                               state->pointer_y;
    
    mouse_report_t hidden_pointer = {.y = mouse_y, .x = MAX_SCREEN_COORD};
    output_mouse_report(&hidden_pointer, state);
    
    // 切换输出目标
    set_active_output(state, output_to);
    
    // 设置新屏幕上的鼠标位置
    state->pointer_x = (direction == LEFT) ? MAX_SCREEN_COORD : MIN_SCREEN_COORD;
    // 按比例缩放 Y 坐标，支持不同分辨率
    state->pointer_y = scale_y_coordinate(output->number, 1 - output->number, state);
}
```

**解析**: 切换逻辑基于绝对坐标系统——鼠标的 HID 描述符被修改为使用绝对坐标（0-32767 范围），设备内部维护指针的绝对位置。当指针超出屏幕边界加 `jump_threshold` 时触发切换。`scale_y_coordinate` 函数处理不同分辨率显示器之间的 Y 坐标映射，确保鼠标在两台电脑间切换时高度一致。切换瞬间，先将鼠标"隐藏"到屏幕边缘（park position），再切换输出并设置新屏幕上的起始位置。

### 5. 鼠标加速算法

实现了基于 2D 移动幅度的鼠标加速：

```c
float calculate_mouse_acceleration_factor(int32_t offset_x, int32_t offset_y) {
    const struct curve {
        int value;
        float factor;
    } acceleration[ACCEL_POINTS] = {
        {2, 1},    // 慢速移动 → 1.0x
        {5, 1.1},
        {15, 1.4},
        {30, 1.9},
        {45, 2.6},
        {60, 3.4},
        {70, 4.0}, // 快速移动 → 4.0x
    };
    
    if (!global_state.config.enable_acceleration)
        return 1.0;
    
    // 计算 2D 移动幅度
    const float magnitude = sqrtf((float)(offset_x * offset_x) + 
                                  (float)(offset_y * offset_y));
    
    // 线性插值找到对应的加速因子
    for (int i = 0; i < ACCEL_POINTS - 1; i++) {
        if (magnitude < acceleration[i + 1].value) {
            const float pos = (magnitude - acceleration[i].value) / 
                              (acceleration[i + 1].value - acceleration[i].value);
            return acceleration[i].factor + 
                   pos * (acceleration[i + 1].factor - acceleration[i].factor);
        }
    }
    return acceleration[ACCEL_POINTS - 1].factor;
}
```

**解析**: 加速曲线有 7 个控制点，慢速移动（幅度 ≤ 2）不加速（1.0x），快速移动（幅度 ≥ 70）加速到 4.0x。使用 2D 欧几里得距离而非单独的 X/Y 分量，使加速感更自然。相邻控制点间使用线性插值平滑过渡。

    queue_init(&state->uart_tx_queue, sizeof(uart_packet_t), UART_QUEUE_LENGTH);
    
    // 启动 Core1
    multicore_reset_core1();
    multicore_launch_core1(core1_main);

    tud_init(BOARD_TUD_RHPORT);

    configure_rx_dma(state);

**解析**: 初始化流程清晰：时钟配置（120MHz 为 PIO USB 提供时钟）→ 加载持久化配置 → 硬件探测 → 外设初始化（UART/USB/DMA）→ 多核启动 → 看门狗。`is_config_mode_active` 利用看门狗寄存器在重启后保留数据的特性（RP2040 datasheet 2.8.1.1），通过在 `scratch[5]` 和 `scratch[6]` 写入 magic word 来标记配置模式。
