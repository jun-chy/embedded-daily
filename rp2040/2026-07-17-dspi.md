DSPi 是一款面向 Raspberry Pi Pico（RP2040）和 Pico 2（RP2350）的全功能数字音频处理固件。它将一块不到一杯咖啡价格的 MCU 开发板变成一台能力出众的 USB 声卡 + 板载 DSP 引擎，支持房间校正、主动分频、参数均衡、时间对齐、响度补偿与耳机串扰消除。项目于 2026 年 1 月创建，截至撰稿时已获得 1119 stars，是 RP2040 生态中近期增长最快、工程完成度最高的音频项目之一。

## 二、项目简介与核心特性一览

DSPi 的设计哲学是"用最便宜的 MCU 做最专业的音频处理"。它通过 USB 接收 16/24-bit PCM 音频，经过一条低延迟的线性 DSP 流水线处理后，以 24-bit S/PDIF、I2S 或 PDM 形式输出。固件充分利用了 RP2040/RP2350 的双核架构与 PIO 硬件，将 DSP 计算与 I/O 编码分离到不同核心，实现零 CPU 开销的数字音频输出。

| 特性 | 说明 |
|------|------|
| **USB 音频接口** | 即插即用（macOS/Windows/Linux/iOS），16/24-bit，44.1/48/96 kHz |
| **多路输出** | RP2350 最多 8 通道 S/PDIF/I2S + 1 路 PDM；RP2040 4 通道 + 1 路 PDM |
| **参数均衡** | 每通道 10 段 PEQ，6 种滤波类型，RP2350 共 110 段、RP2040 共 70 段 |
| **矩阵混音器** | 2 输入 × N 输出，每交叉点独立增益与反相 |
| **响度补偿** | 基于 ISO 226:2003 等响曲线的音量自适应 EQ |
| **耳机串扰** | BS2B 互补滤波 + ITD 头间延时，3 种预设 + 自定义 |
| **音量均衡器** | RMS 软拐膝向上压缩，可选 10ms 预读，-6dBFS 安全限幅器 |
| **双核 DSP** | EQ 处理跨核拆分，Core 1 三模式（PDM/EQ Worker/Idle） |
| **预设系统** | 10 槽完整 DSP 配置存于 Flash，支持启动槽位与批量传输 |
| **运行时改引脚** | 所有输出 GPIO 可运行时重分配，无需重新烧录 |
| **USB 固件升级** | 厂商命令重启进 UF2 bootloader，免按 BOOTSEL |

### 4.2 双核分工

| 核心 | 职责 |
|------|------|
| **Core 0** | USB 通信、音频流接收、DSP 主流水线（主 EQ、串扰、响度、矩阵混音、输出槽 1-2 的 EQ/增益/延时）、控制逻辑、Flash 调度 |
| **Core 1（三模式）** | **PDM 模式**：delta-sigma 调制器（超低音输出启用时）；**EQ Worker 模式**：并行处理高编号输出的 EQ/延时/S/PDIF 转换（PDM 关闭时）；**Idle 模式**：无输出需要时 |

## 五、核心代码深度分析

本节是全文重点。所有代码片段均取自项目真实源码，逐段解析其设计意图与实现细节。

### 5.1 系统初始化：`main()` 与 `core0_init()`

入口函数 `main()` 完成最基本的硬件点亮、超频与初始化，随后进入主循环。主循环以"喂狗 + TinyUSB 任务 + 音频排空 + 延迟 Flash 写入"为核心节奏：

```c
int main(void) {
    // 初始 LED 亮起，表明设备已上电存活
    gpio_init(25); gpio_set_dir(25, GPIO_OUT);
    gpio_put(25, 1);

#if !PICO_RP2350
    // RP2040: VCO 1536 MHz / 4 / 2 = 192 MHz ... 实际此处先设 PLL
    set_sys_clock_pll(1536000000, 4, 2);
#endif

    core0_init();

    // 启用看门狗，8 秒超时 —— 防止主循环卡死导致音频永久中断
    watchdog_enable(8000, 1);

    while (1) {
        watchdog_update();              // 喂狗

        tud_task();                     // TinyUSB 设备任务：枚举、控制传输
        usb_notify_tick();              // 排空设备→主机通知到 EP 0x83
        lg_sound_sync_tick();           // LG Sound Sync 检测（节流后执行）
        dac_hw_mute_tick();             // DAC 硬件静音截止时间检查

        // 排空 USB 音频环形缓冲 —— USB ISR 把原始包推入环形缓冲，
        // 完整 DSP 流水线在主循环上下文执行（而非 USB IRQ 上下文）
        if (active_input_source == INPUT_SOURCE_USB) {
            usb_audio_drain_ring();
        } else {
            usb_audio_flush_ring();     // 非 USB 模式防御性清空
        }
        // ... S/PDIF 输入轮询、延迟 Flash 写入处理 ...
    }
}
```

**逐段解析**：
- `gpio_init(25)` 先点亮板载 LED，是最朴素的"我还活着"信号。
- RP2040 分支调用 `set_sys_clock_pll(1536000000, 4, 2)` 设定 VCO=1536MHz；`core0_init()` 内部最终会重设为 307.2MHz（VCO 1536 / 5 / 1），这一频率的精妙之处在于：它使所有支持采样率（44.1/48/96kHz）下的 PIO 分频器均为整数，消除采样率切换时的时钟毛刺。
- `watchdog_enable(8000, 1)` 设 8 秒看门狗。音频实时系统最怕主循环被 Flash 擦写或异常阻塞卡死——看门狗保证设备能在 8 秒内自动复位恢复。
- 主循环的关键设计：**DSP 流水线不在 USB IRQ 中执行**，而是在主循环上下文通过 `usb_audio_drain_ring()` 排空。这避免了在 USB 中断中做重计算，降低了中断延迟，也让 IRQ 只负责把数据搬运到环形缓冲。

`core0_init()` 则完成电压/频率/外设的精密初始化，尤其值得关注 RP2350 分支对 FPU 的配置：

```c
void core0_init() {
    gpio_init(25); gpio_set_dir(25, GPIO_OUT);

#if PICO_RP2350
    // 启用 Flush-to-Zero 和 Default-NaN，用于音频处理。
    // 防止 SVF/biquad 状态衰减时出现非规格化数的性能惩罚。
    {
        uint32_t fpscr;
        __asm__ volatile("vmrs %0, fpscr" : "=r"(fpscr));
        fpscr |= (1 << 24) | (1 << 25);  // FZ + DN 位
        __asm__ volatile("vmsr fpscr, %0" : : "r"(fpscr));
    }

    // RP2350: 307.2MHz (VCO 1536 / 5 / 1) —— 48kHz 下整数 S/PDIF/I2S 分频器
    vreg_set_voltage(VREG_VOLTAGE_1_15);
    busy_wait_ms(10);
    if (!set_sys_clock_hz(307200000, false)) {
        set_sys_clock_hz(150000000, false);   // 回退安全频率
    }
    dspi_flash_apply_clkdiv();   // Flash 时钟降至 sys_clk/6 ≈ 51.2MHz
#else
    vreg_set_voltage(VREG_VOLTAGE_1_15);
    busy_wait_ms(10);
    set_sys_clock_pll(1536000000, 5, 1);      // 307.2MHz
#endif
    // ...
    bus_ctrl_hw->priority = BUSCTRL_BUS_PRIORITY_DMA_W_BITS | BUSCTRL_BUS_PRIORITY_DMA_R_BITS;
    // [关键修复] USB/SPDIF 必须在 PDM 之前初始化。
    // SPDIF 需要 DMA 通道 0（硬编码）。若 PDM 先初始化，会通过
    // dma_claim_unused_channel() 抢占通道 0，导致 SPDIF panic/崩溃。
    usb_sound_card_init();
    // ...
}
```

**逐段解析**：
- **FZ + DN 位**：这是音频浮点 DSP 的经典优化。IIR 滤波器（biquad/SVF）的状态变量在输入静默时会指数衰减趋近于零，产生大量"非规格化数"（denormal）。ARM Cortex-M 的 FPU 处理非规格化数极慢（可能慢几十倍），会导致音频咔哒声甚至卡顿。设置 FZ（Flush-to-Zero）让过小结果直接归零、DN（Default-NaN）统一 NaN 处理，彻底消除这一性能陷阱。
- `vreg_set_voltage(VREG_VOLTAGE_1_15)` 先升压到 1.15V 再提频——升频前必须先满足电压要求，否则会死机。`busy_wait_ms(10)` 等待电压稳定。
- `bus_ctrl_hw->priority` 设置 DMA 读写优先级，确保音频 DMA 不会被 CPU 取指抢占——这对实时音频至关重要。
- 注释里的"关键修复"揭示了一个真实的踩坑顺序依赖：DMA 通道 0 被 SPDIF 硬编码占用，PDM 若先初始化会动态抢占它导致崩溃。这种初始化顺序约束在多外设嵌入式系统中很常见。

### 5.2 采样率切换：`perform_rate_change()`

采样率切换是音频设备最易出问题的环节。DSPi 的实现体现了"先静音→改参数→重算滤波器→排空旧缓冲→同步重启"的严谨流程：

```c
static void perform_rate_change(uint32_t new_freq) {
    switch (new_freq) { case 44100: case 48000: case 96000: break; default: new_freq = 44100; }

    // 先静音并等待 Core 1 EQ worker 排空，再动滤波器系数或 PIO 分频器。
    // 若无此保护，旧采样率的消费者池缓冲会以新 PIO 位时钟播放约 16ms
    // （可闻的音高偏移 + 重同步咔哒声）。
    prepare_pipeline_reset(PRESET_MUTE_SAMPLES);

    audio_format_48k.sample_freq = new_freq;
    // RP2040/RP2350 均为 307.2MHz 固定时钟 —— 无时钟切换

    // 重置同步
    extern volatile bool sync_started;
    extern volatile uint64_t total_samples_produced;
    sync_started = false;
    total_samples_produced = 0;

    // 预计算标称反馈值并重置控制器（USB 同步反馈环）
    nominal_feedback_10_14 = ((uint64_t)new_freq << 14) / 1000;
    feedback_10_14 = nominal_feedback_10_14;
    reset_usb_feedback_loop();

    dsp_recalculate_all_filters((float)new_freq);   // 重算所有 EQ 系数
    loudness_recompute_pending = true;               // 响度表需重算
    crossfeed_update_pending = true;                 // 串扰系数需重算
    leveller_update_pending = true;                  // 均衡器系数需重算
    pdm_update_clock(new_freq);                      // PDM 时钟更新

    audio_i2s_update_all_frequencies(new_freq);      // 原子更新所有 I2S 实例

    // 若非 SPDIF 锁相预填中，排空所有消费者池（旧采样率音频）并同步重启输出
    if (!spdif_prefilling) {
        complete_pipeline_reset();
    }
}
```

**逐段解析**：
- `switch` 白名单只允许 44100/48000/96000，其余强制降级到 44100——防御性编程，防止 USB 主机发来非法采样率请求。
- `prepare_pipeline_reset(PRESET_MUTE_SAMPLES)` 是核心保护：先注入一段静音样本过渡，并等 Core 1 把当前 EQ 块处理完。注释解释了若跳过这一步，旧速率缓冲在新 PIO 时钟下播放 16ms 会产生可闻音高偏移。
- `nominal_feedback_10_14 = ((uint64_t)new_freq << 14) / 1000`：USB 音频同步反馈值。UAC1 同步端点每帧回报一个 10.14 定点数告诉主机"每毫秒应发多少样本"。这里用 `(freq << 14) / 1000` 计算标称值（如 48000 → 48000×16384/1000 = 786432 个 10.14 单位）。
- 采样率变了，所有依赖 `Fs` 的系数都必须重算：EQ 滤波器（`dsp_recalculate_all_filters`）、响度补偿表、串扰低通/全通、音量均衡器、PDM 时钟。一个都不能漏，否则滤波器特性会错。
- `complete_pipeline_reset()` 排空消费者池中的旧音频再重启——但若正处于 SPDIF 输入锁相预填阶段则跳过，因为预填逻辑会自行在缓冲达 50% 时重启输出。

### 5.3 EQ 滤波器系数计算：`dsp_compute_coefficients()`

这是 DSP 引擎的数学核心。DSPi 在 RP2350 上采用混合 SVF/biquad 架构——低频用 Cytomic SVF（线性梯形积分），高频用传统 TDF2 biquad，兼顾低频精度与计算效率：

```c
void dsp_compute_coefficients(EqParamPacket *p, Biquad *bq, float sample_rate) {
    bool user_bypass = (p->bypass == 1);

    if (user_bypass || is_filter_flat(p) || sample_rate == 0) {
        bq->bypass = true;
        // 设为单位增益直通，零 CPU 开销
        bq->b0 = 1.0f; bq->b1 = 0.0f; bq->b2 = 0.0f; bq->a1 = 0.0f; bq->a2 = 0.0f;
        return;
    }
    bq->bypass = false;

    // 输入校验：Q 限制在 [0.1, 20]，频率限制在 [10, Fs*0.45]
    if (p->Q < 0.1f) p->Q = 0.1f;
    if (p->Q > 20.0f) p->Q = 20.0f;
    if (p->freq < 10.0f) p->freq = 10.0f;
    if (p->freq > sample_rate * 0.45f) p->freq = sample_rate * 0.45f;

    float A = powf(10.0f, p->gain_db / 40.0f);   // 振幅系数（增益 dB 转 linear）

#if PICO_RP2350
    // SVF/biquad 交叉决策 + 路径切换时重置状态
    bool was_svf = bq->use_svf;
    bq->use_svf = (p->freq < (sample_rate / 7.5f));   // 低于 Fs/7.5 用 SVF
    if (was_svf != bq->use_svf) {
        bq->s1 = 0.0f; bq->s2 = 0.0f;
        bq->svic1eq = 0.0f; bq->svic2eq = 0.0f;       // 切换拓扑时清状态防瞬态
    }

    if (bq->use_svf) {
        // SVF 系数（Simper "SvfLinearTrapAllOutputs", Cytomic 2021）
        float g = tanf(3.1415926535f * p->freq / sample_rate);  // 模拟频率预畸变
        float k = 1.0f / p->Q;                                   // 阻尼系数

        switch (p->type) {
            case FILTER_PEAKING:  k = 1.0f / (p->Q * A); break;  // Peak 需调整 k
            case FILTER_LOWSHELF:  g = g / sqrtf(A); break;       // 低频架缩放
            case FILTER_HIGHSHELF: g = g * sqrtf(A); break;       // 高频架缩放
            default: break;
        }

        float sva1_f = 1.0f / (1.0f + g * (g + k));   // 公共分母倒数
        float sva2_f = g * sva1_f;
        float sva3_f = g * sva2_f;

        // 混合系数 m0/m1/m2 决定输出类型（LP/HP/Peak/Shelf...）
        float svm0_f = 0.0f, svm1_f = 0.0f, svm2_f = 0.0f;
        switch (p->type) {
            case FILTER_LOWPASS:   svm0_f = 0.0f; svm1_f = 0.0f;            svm2_f = 1.0f;       break;
            case FILTER_HIGHPASS:  svm0_f = 1.0f; svm1_f = -k;              svm2_f = -1.0f;      break;
            case FILTER_PEAKING:   svm0_f = 1.0f; svm1_f = k*(A*A - 1.0f);  svm2_f = 0.0f;       break;
            case FILTER_LOWSHELF:  svm0_f = 1.0f; svm1_f = k*(A - 1.0f);    svm2_f = A*A - 1.0f; break;
            case FILTER_HIGHSHELF: svm0_f = A*A;  svm1_f = k*(1.0f-A)*A;    svm2_f = 1.0f - A*A; break;
            // ...
        }
        bq->sva1 = sva1_f; bq->sva2 = sva2_f; bq->sva3 = sva3_f;
        bq->svm0 = svm0_f; bq->svm1 = svm1_f; bq->svm2 = svm2_f;
        return;
    }
#endif
    // 传统 RBJ biquad 系数（略，标准 Audio EQ Cookbook 公式）
    float omega = 2.0f * 3.1415926535f * p->freq / sample_rate;
    float sn = sinf(omega); float cs = cosf(omega);
    float alpha = sn / (2.0f * p->Q);
    // ... 按 type 计算 b0~a2 ...
}
```

**逐段解析**：
- `is_filter_flat()` 检测：type 为 FLAT、频率为 0、或 Peak/Shelf 增益接近 0dB（`fabsf < 0.01`）的滤波器都视为平坦，直接 bypass。这很关键——大量未使用的 EQ 段会自动跳过，零 CPU 开销。
- `A = powf(10, gain_db/40)`：注意分母是 40 而非 20，因为 `A` 是振幅的平方根（`A² = 10^(gain/20)`），RBJ 公式的标准约定。
- **SVF vs biquad 决策**：`bq->use_svf = (p->freq < sample_rate / 7.5f)`。在 48kHz 下约 6.4kHz 以下用 SVF。原因是 biquad 在低频时系数量化误差大（极点靠近单位圆），数值精度恶化；而 SVF（State Variable Filter）的线性梯形积分拓扑在低频数值特性优异。这是专业音频 DSP 的成熟做法。
- `g = tanf(π·f/Fs)` 是双线性变换的频率预畸变，将模拟频率映射到数字域。
- SVF 用 `sva1/sva2/sva3`（积分器系数）和 `svm0/svm1/svm2`（输出混合系数）描述。不同滤波类型只需改混合系数，积分核不变——这是 SVF "all outputs from one topology" 的优雅之处。
- 拓扑切换时清零状态（`s1/s2/svic1eq/svic2eq`），避免旧状态在新拓扑下产生瞬态冲击。

### 5.4 滤波处理内循环：`dsp_process_channel_block()`

这是每个音频样本都要跑的热点路径。RP2350 上对 SVF 按滤波类型特化内循环，消除零乘法：

```c
DSP_TIME_CRITICAL
void dsp_process_channel_block(Biquad * __restrict biquads, float * __restrict samples,
                               uint32_t count, uint8_t channel) {
    uint8_t num_bands = channel_band_counts[channel];

    for (int band = 0; band < num_bands; band++) {
        Biquad *bq = &biquads[band];
        if (bq->bypass) continue;          // bypass 段直接跳过

        if (bq->use_svf) {
            // 加载 SVF 系数到局部变量（寄存器驻留）
            float a1 = bq->sva1, a2 = bq->sva2, a3 = bq->sva3;
            float m0 = bq->svm0, m1 = bq->svm1, m2 = bq->svm2;
            float ic1eq = bq->svic1eq, ic2eq = bq->svic2eq;
            float *sp = samples;

            // 按类型特化内循环：消除零乘法
            switch (bq->svf_type) {
                case FILTER_LOWPASS:
                    for (uint32_t i = 0; i < count; i++) {
                        float in = *sp;
                        float v3 = in - ic2eq;
                        float v1 = a1 * ic1eq + a2 * v3
                        float v2 = ic2eq + a2 * ic1eq + a3 * v3;
                        ic1eq = 2.0f * v1 - ic1eq;     // 状态更新
                        ic2eq = 2.0f * v2 - ic2eq;
                        *sp++ = v2;                    // LP 输出 = v2
                    }
                    break;
                case FILTER_HIGHPASS:
                    for (uint32_t i = 0; i < count; i++) {
                        float in = *sp;
                        float v3 = in - ic2eq;
                        float v1 = a1 * ic1eq + a2 * v3;
                        float v2 = ic2eq + a2 * ic1eq + a3 * v3;
                        ic1eq = 2.0f * v1 - ic1eq;
                        ic2eq = 2.0f * v2 - ic2eq;
                        *sp++ = in + m1 * v1 - v2;     // HP = in + m1·v1 - v2
                    }
                    break;
                // Peak/Notch/Allpass、Shelf 各自特化...
                default: // LowShelf/HighShelf
                    for (uint32_t i = 0; i < count; i++) {
                        float in = *sp;
                        float v3 = in - ic2eq;
                        float v1 = a1 * ic1eq + a2 * v3;
                        float v2 = ic2eq + a2 * ic1eq + a3 * v3;
                        ic1eq = 2.0f * v1 - ic1eq;
                        ic2eq = 2.0f * v2 - ic2eq;
                        *sp++ = m0 * in + m1 * v1 + m2 * v2;   // 通用混合输出
                    }
                    break;
            }
            bq->svic1eq = ic1eq;   // 循环结束写回状态
            bq->svic2eq = ic2eq;

        } else {
            // 单精度 TDF2 biquad（高频段）
            float b0 = bq->b0, b1 = bq->b1, b2 = bq->b2;
            float a1 = bq->a1, a2 = bq->a2;
            float s1 = bq->s1, s2 = bq->s2;
            float *sp = samples;

            for (uint32_t i = 0; i < count; i++) {
                float in = *sp;
                float out = b0 * in + s1;          // Transposed Direct Form II
                s1 = b1 * in - a1 * out + s2;
                s2 = b2 * in - a2 * out;
                *sp++ = out;
            }
            bq->s1 = s1;
            bq->s2 = s2;
        }
    }
}
```

**逐段解析**：
- `DSP_TIME_CRITICAL` 宏标记该函数为时间关键，配合链接脚本将其放入 RAM（而非 XIP Flash），避免 Flash 取指延迟影响实时性。
- `__restrict` 关键字告诉编译器 biquads 与 samples 不重叠，允许更激进的优化（消除别名检查）。
- 系数先从结构体加载到局部变量（`a1/a2/a3/m0/m1/m2`），编译器会将其分配到 FPU 寄存器，内循环中无需反复访存。
- **按类型特化**是性能关键：低通输出 `*sp++ = v2`（`m0=m1=0, m2=1`），若用通用公式 `m0*in + m1*v1 + m2*v2` 会多做两次零乘法。特化后每个类型只算必要的乘加。对 110 段 EQ、每段处理数百样本，这种优化省下的周期相当可观。
- SVF 核心三式：`v3 = in - ic2eq`、`v1 = a1·ic1eq + a2·v3`、`v2 = ic2eq + a2·ic1eq + a3·v3`，然后状态更新 `ic1eq = 2·v1 - ic1eq`（梯形积分的 Z 变换形式）。这是 Cytomic 的线性梯形 SVF，数值稳定且对音频冲激响应友好。
- TDF2（Transposed Direct Form II）biquad：`out = b0·in + s1`，`s1 = b1·in - a1·out + s2`。转置直接型 II 对系数量化噪声更鲁棒，是音频 IIR 的首选结构。

### 5.5 RP2040 定点乘法：`fast_mul_q28()`

RP2040 没有硬件 FPU，DSPi 用 Q28 定点 + 手写优化乘法。标准 32×32→64 乘法在 Cortex-M0+ 上需要库函数调用，这里拆成 4 次 16×16→32 硬件乘法：

```c
#if !PICO_RP2350
DSP_TIME_CRITICAL int32_t fast_mul_q28(int32_t a, int32_t b) {
    int32_t ah = a >> 16;           // 高 16 位（带符号）
    uint32_t al = a & 0xFFFF;       // 低 16 位（无符号）
    int32_t bh = b >> 16;
    uint32_t bl = b & 0xFFFF;

    int32_t high = ah * bh;         // 高×高
    int32_t mid1 = ah * bl;         // 高×低
    int32_t mid2 = al * bh;         // 低×高

    return (high << 4) + ((mid1 + mid2) >> 12);
}
#endif
```

**逐段解析**：
- Q28 定点：1.0 表示为 `1 << 28 = 268435456`。两个 Q28 数相乘结果为 Q56，需要右移 28 位回到 Q28。
- 这里把 32 位拆成高低 16 位各做一次硬件乘法（Cortex-M0+ 的 `MUL` 指令是 32×32→32 单周期但只取低 32 位）。`high<<4` 对应高×高贡献（`ah·bh` 是 Q32×Q32=Q64，但只取其有效部分，左移 4 调整到 Q28 量纲），`(mid1+mid2)>>12` 是交叉项贡献。
- 实际上 RP2040 的最终内循环更进一步——`dsp_process_channel_block` 在 RP2040 上由 `dsp_process_rp2040.S` 手写汇编实现（见文件末尾 `extern` 声明），榨干每一点周期预算。这是 RP2040 能在 307MHz 跑 70 段 EQ 的根本原因。

### 5.6 BS2B 耳机串扰消除：`crossfeed.c`

耳机听音时左右声道完全隔离，缺乏扬声器聆听时的声学串扰（头影 + 头间延时），导致声场不自然。BS2B（Bauer Stereophonic-to-Binaural）通过把对侧信号经低通滤波与延时后混入本侧来模拟这种串扰。DSPi 采用**互补滤波**设计保证单声道信号无染色直通：

```c
void crossfeed_compute_coefficients(CrossfeedState *state, const CrossfeedConfig *config, float sample_rate) {
    // ... 取 fc 与 feed_db（预设或自定义）...

    // 用互补约束计算串扰增益 G：
    //   feed_db 是直通与串扰在 DC 的电平差：20·log10(direct_dc / cross_dc)
    //   互补约束：direct_dc + cross_dc = 1
    //   => cross_dc = 1/(1+level_ratio) = G, direct_dc = 1 - G
    float level_ratio = powf(10.0f, feed_db / 20.0f);
    float G = 1.0f / (1.0f + level_ratio);

    // 单极点 IIR 低通：H(z) = G·(1-x)/(1 - x·z^-1)，x = exp(-2π·Fc/Fs)
    float x = expf(-2.0f * 3.1415926535f * fc / sample_rate);
    float lp_a0_f = G * (1.0f - x);   // DC 增益 = G
    float lp_b1_f = x;

    // 一阶全通滤波器实现头间延时 ITD
    // 低通本身在 DC 已有相位延时 τ_lp = x/((1-x)·Fs)
    // 剩余延时由全通提供：群延时@DC = (1-a)/(1+a) 样本
    //   解 a = (1-D)/(1+D)，D = 剩余延时样本数
    float ap_a_f;
    if (config->itd_enabled) {
        float lp_delay_sec = x / ((1.0f - x) * sample_rate);
        float remaining_sec = CROSSFEED_ITD_SEC - lp_delay_sec;   // ITD 目标 ~220µs
        if (remaining_sec > 0.0f) {
            float D = remaining_sec * sample_rate;
            ap_a_f = (1.0f - D) / (1.0f + D);
        } else {
            ap_a_f = 1.0f;   // 低通已提供足够延时
        }
    } else {
        ap_a_f = 1.0f;       // ITD 关闭：全通退化为直通
    }
    // ... 存入 state（RP2350 float / RP2040 Q28）...
}
```

处理函数体现了"直接 = 输入 - 自身低通，输出 = 直接 + 对侧全通(低通)"的互补结构：

```c
DSP_TIME_CRITICAL
void crossfeed_process_stereo(CrossfeedState *state, float *left, float *right) {
    float in_L = *left;
    float in_R = *right;

    // 低通滤波两通道：cross = G × L(z) × input
    float lp_out_L = state->lp_a0 * in_L + state->lp_b1 * state->lp_state_L;
    float lp_out_R = state->lp_a0 * in_R + state->lp_b1 * state->lp_state_R;
    state->lp_state_L = lp_out_L;
    state->lp_state_R = lp_out_R;

    // 全通滤波串扰信号实现 ITD（TDF2 一阶全通）
    float ap_out_L = state->ap_a * lp_out_L + state->ap_state_L;
    state->ap_state_L = lp_out_L - state->ap_a * ap_out_L;
    float ap_out_R = state->ap_a * lp_out_R + state->ap_state_R;
    state->ap_state_R = lp_out_R - state->ap_a * ap_out_R;

    // 互补混合：
    //   direct = input - own_lowpass（未延时的互补分量）
    //   output = direct + allpass(opp_lowpass)（对侧延时串扰）
    *left  = (in_L - lp_out_L) + ap_out_R;
    *right = (in_R - lp_out_R) + ap_out_L;
}
```

**逐段解析**：
- **互补约束的妙处**：`direct = in - lp(in)`，`output = direct + ap(lp(对侧))`。对单声道信号（L=R），`lp_L = lp_R`，直流下 `direct + cross = (1-G) + G = 1`，所以单声道以单位增益直通、无染色。对高频硬声像（低通→0），`direct ≈ in`，高频不串扰，保持立体声分离度。这正好符合人耳对低频串扰敏感、高频定位精确的物理特性。
- `G = 1/(1+10^(feed/20))`：feed_db 越大，G 越小，串扰越弱、直通越强。4.5dB 时 G≈0.373、直通 0.627。
- **ITD 全通**：头间延时约 220µs（声波绕头传播 60° 扬声器摆位的物理延时）。低通本身在 DC 已有相位延时，剩余部分由一阶全通补足。`a=1` 时全通退化为 `y=x`（无延时），巧妙地把"是否启用 ITD"变成一个系数切换。
- 处理函数对左右声道对称处理，`ap_out_R` 混入左声道、`ap_out_L` 混入右声道——模拟对侧声音绕过头部到达本侧耳。

### 5.7 ISO 226 响度补偿：`loudness.c`

人耳对不同频率的敏感度随声压级变化（等响曲线）。低音量下低频和高频主观感知明显衰减。DSPi 依据 ISO 226:2003 标准计算补偿增益，并预计算 91 级音量步的双缓冲系数表实现无毛刺切换：

```c
// ISO 226:2003 SPL 计算（标准公式 1-2）
static float iso226_spl(float Tf, float af, float Lu, float phon) {
    // 阈值项：(0.4 × 10^((Tf+Lu)/10 - 9))^af
    float B = 0.4f * powf(10.0f, (Tf + Lu) / 10.0f - 9.0f);
    float threshold = powf(B, af);

    // Af = 声压相关项 + 阈值项
    float Af = 4.47e-3f * (powf(10.0f, 0.025f * phon) - 1.15f) + threshold;
    if (Af < 1e-10f) Af = 1e-10f;   // 防 log(0)

    // Lp = (10/αf)·log10(Af) - Lu + 94
    return (10.0f / af) * log10f(Af) - Lu + 94.0f;
}

// 计算某频率在某音量步的补偿增益（dB）
static float loudness_compensation_db(float Tf, float af, float Lu,
                                       float ref_spl, float effective_phon,
                                       float intensity_pct) {
    if (effective_phon >= ref_spl) return 0.0f;   // 参考电平无需补偿

    float spl_ref = iso226_spl(Tf, af, Lu, ref_spl);        // 参考声压下该频 SPL
    float spl_eff = iso226_spl(Tf, af, Lu, effective_phon); // 降低后该频 SPL

    // 补偿 = [该频 SPL 实际变化] - [1kHz 平坦衰减量]
    float flat_change = effective_phon - ref_spl;   // 负值（音量降低）
    float freq_change = spl_eff - spl_ref;          // 该频 SPL 实际变化
    float compensation = freq_change - flat_change; // 正值 = 需要提升

    compensation *= (intensity_pct / 100.0f);       // 强度缩放
    return compensation;
}
```

双缓冲重算确保音量变化时系数表无缝切换：

```c
void loudness_recompute_table(float ref_spl, float intensity_pct, float sample_rate) {
    // 写入非活动缓冲
    uint8_t write_buf = 1 - active_buf;

    static const float shelf_freq[2] = { 200.0f, 6000.0f };  // 低/高频架中心
    static const float shelf_Q = 0.707f;

    for (int vol_idx = 0; vol_idx < LOUDNESS_VOL_STEPS; vol_idx++) {
        float vol_db = (float)(vol_idx - 60);             // 索引 0 = -60dB, 60 = 0dB
        float effective_phon = ref_spl + vol_db;
        if (effective_phon < 20.0f) effective_phon = 20.0f;
        if (effective_phon > ref_spl) effective_phon = ref_spl;

        // 50Hz 处低频架增益（ISO 226 表 1: αf=0.432, Lu=-15.9, Tf=44.0）
        float low_gain_db = loudness_compensation_db(
            ISO_50_TF, ISO_50_AF, ISO_50_LU, ref_spl, effective_phon, intensity_pct);
        // 10kHz 处高频架增益（ISO 226: αf=0.271, Lu=-10.7, Tf=13.9）
        float high_gain_db = loudness_compensation_db(
            ISO_10K_TF, ISO_10K_AF, ISO_10K_LU, ref_spl, effective_phon, intensity_pct);

        // 计算低/高频架 biquad/SVF 系数
        compute_shelf_coeffs(shelf_freq[0], shelf_Q, low_gain_db, 0, sample_rate,
                             &loudness_tables[write_buf][vol_idx][0]);
        compute_shelf_coeffs(shelf_freq[1], shelf_Q, high_gain_db, 1, sample_rate,
                             &loudness_tables[write_buf][vol_idx][1]);
    }

    // 原子切换：更新活动表指针
    active_buf = write_buf;
    loudness_active_table = loudness_tables[active_buf];
}
```

**逐段解析**：
- `iso226_spl()` 实现标准 ISO 226:2003 公式。`Tf/af/Lu` 是标准附录表 1 中各频率的常数（50Hz: `Tf=44, αf=0.432, Lu=-15.9`；10kHz: `Tf=13.9, αf=0.271, Lu=-10.7`）。`phon` 是响度级，函数返回该频率在该响度级下的声压级 SPL。
- 补偿逻辑精髓在 `compensation = freq_change - flat_change`：`flat_change` 是 1kHz 处的纯音量衰减（线性），`freq_change` 是目标频率 SPL 的实际变化。两者之差就是"该频率比 1kHz 多衰减了多少"，即需要补偿提升的量。这保证补偿后各频率相对响度与参考电平一致。
- **双缓冲**：`write_buf = 1 - active_buf` 写入非活动缓冲，全部算完后 `active_buf = write_buf` 原子切换指针。这样音频处理线程读活动表时永远不会读到半完成的数据，实现无毛刺更新。
- 只需 2 个频率点（50Hz 低频架 + 10kHz 高频架）即可覆盖人耳最敏感的补偿区域，兼顾精度与 CPU。
- 注释特别指出修正了旧版本中 `Lu` 符号翻转和 10kHz 处 `αf` 偏差约 10% 的 bug——这类等响曲线常数错误会让补偿量偏差 2.5 倍，可见作者对标准实现的严谨。

### 5.8 主音频流水线：`process_input_block()`

这是整个 DSP 引擎的调度中枢，按 PASS 1~7 顺序处理每个音频块。函数标记 `__not_in_flash_func` 确保运行在 RAM 中：

```c
void __not_in_flash_func(process_input_block)(uint32_t sample_count) {
    uint32_t packet_start = time_us_32();

    // 为各 S/PDIF 输出获取生产者缓冲
    struct audio_buffer* audio_buf[4] = {NULL, NULL, NULL, NULL};
    if (producer_pool_1) audio_buf[0] = take_audio_buffer(producer_pool_1, false);
    // ...

    uint32_t sample_rate_hz = audio_state.freq;
    float preset_mute_gain = update_preset_mute_envelope(sample_count, sample_rate_hz);

    // ===== 复合增益斜坡：host音量 × 预设静音包络 × 主音量 =====
    bool muted = (s_uac_mute && host_active) || s_user_mute;
    float vol_mul_target = muted ? 0.0f : (float)audio_state.vol_mul * inv_32768;
    vol_mul_target *= preset_mute_gain;
    float vol_mul_master_target = vol_mul_target * master_volume_linear;
    float vol_mul_master_step = (vol_mul_master_target - vol_mul_master_start) / sample_count;
    // 稳态时 step=0，回落到常量增益快速路径，无额外逐样本开销

    // ========== PASS 2: 主 EQ（块处理）==========
    if (!is_bypassed) {
        if (!channel_bypassed[CH_MASTER_LEFT])
            dsp_process_channel_block(filters[CH_MASTER_LEFT], buf_l, sample_count, CH_MASTER_LEFT);
        if (!channel_bypassed[CH_MASTER_RIGHT])
            dsp_process_channel_block(filters[CH_MASTER_RIGHT], buf_r, sample_count, CH_MASTER_RIGHT);
    }

    // ========== PASS 2.5: 音量均衡器 ==========
    if (!leveller_bypassed)
        leveller_process_block(&leveller_state, &leveller_coeffs,
                               (const LevellerConfig *)&leveller_config,
                               buf_l, buf_r, sample_count);

    // ========== PASS 3: 串扰 + 主峰值表 ==========
    for (uint32_t i = 0; i < sample_count; i++) {
        float ml = buf_l[i], mr = buf_r[i];
        float abs_ml = fabsf(ml); if (abs_ml > peak_ml) peak_ml = abs_ml;  // 峰值检测
        if (do_crossfeed) {
            crossfeed_process_stereo(&crossfeed_state, &ml, &mr);
            buf_l[i] = ml; buf_r[i] = mr;
        }
    }

    // ========== PASS 4: 矩阵混音（输出优先，块处理）==========
    for (int out = 0; out < NUM_OUTPUT_CHANNELS; out++) {
        if (!matrix_mixer.outputs[out].enabled) {
            memset(buf_out[out], 0, sample_count * sizeof(float));  // 禁用输出置零省 CPU
            continue;
        }
        // 每输出加载一次交叉点系数（非每样本）
        float gain_l = xp_l->phase_invert ? -xp_l->gain_linear : xp_l->gain_linear;
        float gain_r = xp_r->phase_invert ? -xp_r->gain_linear : xp_r->gain_linear;

        if (gain_l != 0.0f && gain_r != 0.0f)
            for (uint32_t i = 0; i < sample_count; i++)
                dst[i] = buf_l[i] * gain_l + buf_r[i] * gain_r;
        // ... 单通道分支 ...
    }

    // ========== PASS 5-7: 每输出 EQ + 增益 + 延时 + 输出 ==========
    if (core1_mode == CORE1_MODE_EQ_WORKER) {
        // --- 双核路径：Core 1 处理输出 2-7 的 EQ+延时+SPDIF ---
        core1_eq_work.sample_count = sample_count;
        core1_eq_work.vol_mul_start = vol_mul_master_start;
        core1_eq_work.vol_mul_step  = vol_mul_master_step;
        core1_eq_work.work_done = false;
        __dmb();            // 数据内存屏障：确保 Core 1 看到完整工作包
        core1_eq_work.work_ready = true;
        __sev();            // 发送事件：唤醒 Core 1 的 WFE 等待

        // Core 0 同时处理输出 0-1
        for (int out = 0; out < CORE1_EQ_FIRST_OUTPUT; out++) {
            // EQ + 增益斜坡 + 延时 + S/PDIF 编码 ...
        }
    }
    // ...
}
```

**逐段解析**：
- `__not_in_flash_func` 将函数放入 RAM 段，避免 XIP（eXecute In Place）从 Flash 取指的延迟波动——音频回调对执行时间确定性要求极高。
- **增益斜坡**（`vol_mul_master_step`）：音量变化时不是硬切换（会产生咔哒声），而是跨 `sample_count` 样本线性斜坡过渡。稳态时 step=0，跳入常量增益快速路径，零额外开销。这是专业音频软件消除 zipper noise 的标准手法。
- **PASS 4 矩阵混音的输出优先策略**：外层按输出遍历、内层按样本遍历，每输出只加载一次交叉点系数。若改为样本优先则每样本都要重载系数，cache 不友好。
- **双核协作**：Core 0 通过共享结构 `core1_eq_work` 派发工作给 Core 1。`__dmb()`（数据内存屏障）保证 Core 1 看到完整的工作包字段后才看到 `work_ready=true`；`__sev()`（Send Event）唤醒 Core 1 的 `WFE`（Wait For Event）等待。两核共享相同的增益斜坡参数，保证分配到不同核的输出相位对齐。这是无锁双核通信的典范。

### 5.9 关键配置常量：`config.h`

```c
#define ENABLE_SUB 1

#define PICO_AUDIO_SPDIF_PIN   6    // S/PDIF 1 (Out 1-2)
#define PICO_SPDIF_PIN_2       7    // S/PDIF 2 (Out 3-4)
#define PICO_PDM_PIN           10   // PDM 超低音 (Out 9)

#define FILTER_SHIFT 28             // Q28 定点

#define PDM_OVERSAMPLE 256          // 256× 过采样
#define PDM_DMA_BUFFER_SIZE 2048
#define PDM_DITHER_MASK 0x1FF       // TPDF 抖动幅度
#define PDM_LEAKAGE_SHIFT 16        // 漏积分器（DC 保护）~1.4s 时间常数

#define AUDIO_BUFFER_COUNT    8     // 每实例生产者缓冲
#define SPDIF_CONSUMER_BUFFER_COUNT 16
#define AUDIO_BUFFER_SAMPLES  192

#if PICO_RP2350
#define MAX_DELAY_SAMPLES 2048   // 42ms @ 48kHz
#else
#define MAX_DELAY_SAMPLES 1024   // 21ms @ 48kHz（RP2040 SRAM 预算）
#endif

#define SUB_ALIGN_SAMPLES (SPDIF_BUFFER_SAMPLES - PDM_BUFFER_SAMPLES)  // 128，PDM 路径延时补偿
```

**逐段解析**：
- `FILTER_SHIFT 28`：Q28 定点，1.0 = 2^28。28 而非 31 是为了留出 4 位头部空间防止乘法溢出。
- `PDM_LEAKAGE_SHIFT 16`：delta-sigma 调制器的漏积分器每样本右移 16 位，等效时间常数约 1.4 秒（48kHz 下 65536 样本），防止 DC 偏移累积烧低音炮——长到不影响低频，短到能缓慢释放 DC。
- `MAX_DELAY_SAMPLES` 双平台差异：RP2040 仅 264KB SRAM 且需 copy_to_ram，延时线只能给 1024 样本（21ms）；RP2350 SRAM 充裕给到 2048（42ms）。
- `SUB_ALIGN_SAMPLES = 128`：S/PDIF 路径与 PDM 路径的缓冲深度差。PDM DMA 缓冲比 S/PDIF 消费者池浅 128 样本，软件给 PDM 输出额外补 128 样本延时，使两路径输出样本对齐——这是多输出低音管理的延时补偿关键。

## 六、API 使用指南

DSPi 通过 USB 厂商接口（Interface 2，Windows；Interface 0，macOS）用控制传输配置 DSP。所有命令以单字节 `bRequest` 区分，`wValue/wIndex` 携带参数。

### 6.1 Python 示例（pyusb）

```python
import usb.core, usb.util, struct

# 查找 DSPi 设备（VID/PID 见 usb_descriptors.c）
dev = usb.core.find(idVendor=0x2E8A, idProduct=0xDSPi)
# WinUSB 接口号
IFACE = 2

def vendor_out(bRequest, wValue, data, wIndex=0):
    """发送厂商 OUT 命令"""
    dev.ctrl_transfer(
        0x40,            # bmRequestType = Vendor, OUT, Device
        bRequest, wValue, wIndex | (IFACE << 8),
        data)

def vendor_in(bRequest, wValue, length, wIndex=0):
    """读取厂商 IN 响应"""
    return dev.ctrl_transfer(
        0xC0,            # bmRequestType = Vendor, IN, Device
        bRequest, wValue, wIndex | (IFACE << 8),
        length)

# 设置主音量为 -10 dB（0xD2 = REQ_SET_MASTER_VOLUME）
vendor_out(0xD2, 0, struct.pack('<f', -10.0))

# 启用耳机串扰（0x5E = REQ_SET_CROSSFEED）
vendor_out(0x5E, 0, [1])

# 设置串扰预设为 Default（0 = Default, 0x60 = REQ_SET_CROSSFEED_PRESET）
vendor_out(0x60, 0, [0])

# 读取通道 0 峰值表（0x50 = REQ_GET_STATUS, wValue=0）
peaks = vendor_in(0x50, 0, 4)
ch01 = struct.unpack('<I', peaks)[0]

# 设置 EQ 滤波器：通道 0、频段 2、Peaking、1kHz、Q=0.707、+3dB
# 0x42 = REQ_SET_EQ_PARAM，16 字节包
eq_pkt = struct.pack('<BBBBfff', 0, 2, 1, 0, 1000.0, 0.707, 3.0)
vendor_out(0x42, 0, eq_pkt)

# 保存到预设槽 3（0x90 = REQ_PRESET_SAVE, wValue=slot）
vendor_in(0x90, 3, 1)

# 重启进 bootloader 升级固件（0xF0 = REQ_ENTER_BOOTLOADER）
vendor_in(0xF0, 0, 1)
```

## 七、编译与部署

### 7.1 环境搭建

| 工具 | 版本要求 | 用途 |
|------|----------|------|
| CMake | ≥ 3.13 | 构建系统 |
| Arm GNU Toolchain | arm-none-eabi-gcc | 交叉编译 |
| Python 3 | 任意 | Pico SDK 脚本 |
| Git | 任意 | 克隆与子模块 |

### 7.2 编译配置

```bash
# 1. 递归克隆（含 Pico SDK 与 pico-extras 子模块）
git clone --recursive https://github.com/WeebLabs/DSPi.git
cd DSPi
# 若已非递归克隆：
# git submodule update --init --recursive

# 2a. 编译 RP2040（标准 Pico）
mkdir build-rp2040 && cd build-rp2040
cmake -DPICO_BOARD=pico -DPICO_EXTRAS_PATH=../firmware/pico-extras ../firmware
make -j4
# 产物：DSPi/DSPi.uf2

# 2b. 编译 RP2350（Pico 2）
mkdir build-rp2350 && cd build-rp2350
cmake -DPICO_BOARD=pico2 -DPICO_EXTRAS_PATH=../firmware/pico-extras ../firmware
make -j4
```

CMake 构建系统通过 `PICO_RP2350` 宏自动选择平台路径：RP2350 走浮点 + 混合 SVF/biquad；RP2040 走 Q28 定点 + 手写汇编内循环。`PICO_BOARD` 决定内存布局、boot2 与引脚默认值。

### 7.3 烧录步骤

1. 按住 Pico 的 **BOOTSEL** 键插入电脑，出现 `RPI-RP2`（或 `RP2350`）U 盘。
2. 将编译出的 `DSPi.uf2` 拖入该盘，设备自动重启并枚举为 "Weeb Labs DSPi" 音频设备。
3. 下载 DSPi Console 控制软件配置 DSP。
4. 后续升级可免按键：已运行的 DSPi 收到 `REQ_ENTER_BOOTLOADER`(0xF0) 即重启进 UF2 bootloader。

## 八、项目亮点与适用场景

**工程亮点**：
- **混合 SVF/biquad 架构**：低频 SVF 保证数值精度、高频 biquad 保证效率，是专业音频 DSP 的最佳实践。
- **双核无锁协作**：Core 0/Core 1 通过 `__dmb()+__sev()+WFE` 共享工作包，零中断开销实现 EQ 并行。
- **Q28 定点 + 手写汇编**：RP2040 无 FPU 仍跑 70 段 EQ，靠的是 `dsp_process_rp2040.S` 汇编内循环榨取每一点算力。
- **互补滤波串扰**：BS2B 互补设计保证单声道无染色，数学上优雅。
- **双缓冲响度表**：91 级音量预计算系数表 + 原子指针切换，音量变化无毛刺。
- **运行时引脚重分配**：所有输出 GPIO 可运行时改，适配自定义 PCB，免重烧录。

**适用场景**：
- **主动分频音箱**：2~4 路电子分频，每通道独立 EQ 与延时对齐。
- **耳机放大器**：串扰消除 + 响度补偿 + 参数 EQ 一站式。
- **家庭影院低音管理**：PDM 直出超低音，S/PDIF/I2S 主声道，自动延时补偿。
- **USB DAC/DSP 前级**：即插即用 USB 声卡 + 房间校正 EQ。
- **教学与研究**：真实可读的音频 DSP 固件范例，涵盖定点/浮点、滤波器拓扑、USB 音频、双核调度。

## 九、总结

DSPi 是 RP2040/RP2350 生态中难得的"完成度 + 工程深度"双高的音频项目。它不仅把一块几十元的开发板变成了专业级 USB DSP，更在源码层面展示了大量值得学习的嵌入式工程技艺：从 FPU 非规格化数处理、DMA 通道顺序依赖、看门狗保护，到混合 SVF/biquad 滤波器拓扑、Q28 定点汇编优化、双核无锁通信、互补滤波串扰、ISO 226 响度补偿双缓冲。对于想深入数字音频 DSP 与 RP2040 双核编程的开发者，这是一个极佳的实战参考。项目 2026 年 1 月才创建，半年内获得 1119 stars 且仍在每日更新，值得关注与跟踪。
